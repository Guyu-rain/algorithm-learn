# 回溯算法解决所有排列/组合/子集问题
先回顾以下回溯算法：
```python
# 传入已经选择过的 和 备选的
def backtrack(路径, 选择列表):
    if 路径满足条件:
        result.append(路径)
        return
    
    for 选择 in 选择列表:
        # 做选择
        选择列表中去掉这个选择
        路径.append(选择)
        backtrack(路径, 选择列表)
        路径.remove(选择)
        选择列表中重新加入这个选择
```
从备选集合和可选两个方面来看，排列、组合、子集问题分成三种：
1. 元素无重复 + 不可复选
2. 元素可重复 + 不可复选
3. 元素不重复 + 可以复选
- 元素可重复 + 可以复选 这种没有意义。都可以复选了，重复与否都没有意义了。

不管是这三种哪一种的排列/组合/子集问题（一共3x3=9种），或者其他的变体，本质都是穷举所有解，而且这个解结构必然**呈现树状结构**，所以需要合理使用回溯框架 + 题目限制，来解决所有问题。这个树结构可以分成：
- 组合 / 子集树（组合和子集问题是等价的）
- 排列树

## 子集
子集最重要的是不出现重复元素：
我们通过**保证元素之间的相对顺序不变**防止出现重复的子集。
子集树有如下特点：
**如果将根节点视作第 0 层，将每个节点和根节点之间的树枝上的元素作为该节点的值，那么第 n 层的所有节点就是大小为 n 的所有子集。**
```text
                  [ ]
            /      |      \
          [1]     [2]     [3]
         /   \     |
      [1,2] [1,3] [2,3]
        |
     [1,2,3]
```
注意，这颗树的树枝上表示元素的选择，从左向右保持着相对顺序不变。
```C++
// res 就是答案
vector<vector<int>> res;
// 记录回溯算法的递归路径
vector<int> track;

void backtrack(vector<int>& nums, int start) {
    // 前序位置，每个节点都是一个子集
    res.push_back(track);

    // 回溯算法标准框架
    // 需要保证元素之间的相互顺序
    for (int i = start; i < nums.size(); i++) {
        // 做选择
        track.push_back(nums[i]);
        // 通过参数 start 控制树的遍历，也就是控制元素之间的相对关系
        backtrack(nums, i + 1);
        // 撤销选择
        track.pop_back();
    }
}
```
是如何保证元素相对顺序不变的？我们用了一个多余的信息 `start参数`，保证了元素之间顺序不变。

## 组合
组合问题可以很轻松的转化成子集问题：**大小为`k`的组合 等价于 所有大小为`k`的子集**，也就是要第`k`层子集树上的结果。
```C++
vector<vector<int>> res;
// 记录回溯算法的递归路径
vector<int> track;

void backtrack(vector<int>& nums, int start, int k) {
    // 第 k 层
    if (k == track.size()) {
        res.push_back(track);
        return;
    }

    for (int i = start; i < nums.size(); i++) {
        // 做选择
        track.push_back(nums[i]);
        // 进入下一个节点
        backtrack(nums, i + 1, k);
        // 撤销选择
        track.pop_back();
    }
}
```
可以做一个简单的剪枝，当剩余元素`nums.size()-i` < `k-track.size()` 时，必然不可能产生一个大小为 k 的集合 `track`，因此直接`return`即可

## 排列
排列组成的树是**排列树**。排列数的特点是不需要限制元素之间的相对位置。
需要限制的是，选过元素的不能再选，需要通过一个`used`数组来标记哪些元素已经备选过。
```C++
// 存储所有排列结果的列表
vector<vector<int>> res;
// 记录回溯算法的递归路径
vector<int> track;
// track 中的元素会被标记为 true
vector<bool> used;

void backtrack(vector<int>& nums) {
    if (track.size() == nums.size()) {
        res.push_back(track);
        return;
    }

    for (int i = 0; i < nums.size(); i++) {
        // 先判断是否可以选，然后做选择
        if (used[i]) {
            continue;
        }
        // 做选择
        used[i] = true;
        track.push_back(nums[i]);
        // 进入下一层回溯树
        backtrack(nums);
        // 撤销选择
        track.pop_back();
        used[i] = false;
    }
}
```
如果要 `k` 个元素组成的，那么添加一个参数，然后改一下判断条件即可：
```C++
void backtrack(vector<int>& nums, int k) {
    if (track.size() == k) {
        res.push_back(track);
        return;
    }

    ...
}
```
排列不需要保证顺序不变，因此没有`start参数`，但是需要保证选过的不再选，因此需要`used[]`。

## 子集 / 集合 （元素可重，不可复选）
### [LeetCode90.子集II](https://leetcode.cn/problems/subsets-ii/)
重复元素的处理，由于我们依靠元素相对关系不能改变来处理子集 / 集合问题，因此第一步需要排序（保证数组有序），否则无法剪枝。

考察一个简单的例子：`nums = {1, 2, 2'}`，这里用`2'`标记有两个2。考虑回溯法，能够形成如下的树：
```
level 0:                        []
level 1:       [1]              [2]         [2']❌
level 2: [1, 2]  [1, 2']❌      [2, 2']
...
```
其中，❌表示重复子集，应该被删除。通过观察可以发现，如果前一个选择和后一个选择相同`nums[i] == nums[i - 1]`，应该只留下第一个树枝(`i == start`)，后面的直接`continue`。如下：
```C++
vector<vector<int>> res;
vector<vector<int>> track;
// 由于通过顺序保证不会重复选择，不需要 used / visited
vector<vector<int>> subsetsWithDup(vector<int>& nums) {
    // 要保证 nums 有序
    std::sort(nums.begin(), nums.end());
    backtrack();
    return res;
}

void backtrack(vector<int>& nums, int start) {
    // 每个节点都是一个答案
    res.push_back(track);

    for (int i = start; i < nums.size(); i++) {
        // 如果这个树枝和前面一个数相同，而且这个数不是第一次出现，剪掉 💡
        // 剪枝逻辑：值相同的相邻树枝，只遍历第一条 
        // 注意：i == start，表示刚新进入一层，此时 nums[i] == nums[i - 1]是和上一层相同，这是被允许的
        if (i > start && nums[i] == nums[i - 1]) {
            continue;
        }
        // 做选择
        track.push_back(nums[i]);
        // 进入下一层节点
        backtrack(nums, i + 1);
        // 撤销选择
        track.pop_back();
    }
}
```

### [LeetCode40.组合总和](https://leetcode.cn/problems/combination-sum-ii/)
本质上还是一个组合，组合树与子集树相同，都是通过限定元素之间的相对位置来规避重复。本题还有一个和的约束，之前的组合是通过树高（集合大小）来剪枝，本题应该维护一个路径和，来剪枝。
还有一个剪枝：如果此时路径和已经超过了 `target`，直接 return。这个剪枝不做没有逻辑错误。
```C++
int sum = 0;
vector<int> track;
vector<vector<int>> res;
vector<vector<int>> combinationSum2(vector<int>& candidates, int target) {
    std::sort(candidates.begin(), candidates.end());
    backtrack(candidates, target, 0);
    return res;
}

void backtrack(vector<int>& nums, int target, int start) {
    // base case, 达到目标和，找到符合条件的组合
    if (sum == target) {
        res.push_back(track);
        return;
    }
    // base case, 剪枝，超过目标和，直接结束
    if (sum > target) {
        return;
    }

    for (int i = start; i < nums.size(); i++) {
        // 剪枝逻辑：在同一层中，出现相同元素，只取第一次
        // 第一次指的是 i == start
        // 注意：i == start 表示这一层第一次出现某元素
        if (i > start && nums[i - 1] == nums[i]) {
            continue;
        }

        // 做选择
        track.push_back(nums[i]); sum += nums[i];
        // 进入下一层
        backtrack(nums, target, i + 1);
        // 撤销选择
        track.pop_back(); sum -= nums[i];
    }
}
```

## 排列（元素可重，不可复选）
### (LeetCode47.全排列II)[https://leetcode.cn/problems/permutations-ii/]
排列的考虑比较困难，我第一次考虑，还是从排列树的角度出发，发现如果一个节点相邻的树枝，如果两个树枝相同，剪枝掉后者，是能够得出答案的。这种考虑，需要记录前面一个树枝的值是什么。如下：
```C++
vector<bool> used = vector<bool>(N, false);

void backtrack(vector<int>& nums, list<int>& track) {
    if (track.size() == nums.size()) {
        res.push_back(vector<int>(track.begin(), track.end()));
        return;
    }

    // 需要记录前面一个树枝上是什么值，随便初始化成一个不可能到的值
    int prevNum = INT_MIN;
    for (int i = 0; i < nums.size(); i++) {
        if (used[i]) {
            continue;
        }
        if (prevNum == nums[i]) {
            continue;
        }
        
        // 做选择
        track.push_back(nums[i]);
        used[i] = true;
        // 记录这条树枝上的值，供下一次进入循环（下一个树枝）剪枝
        prevNum = nums[i];

        backtrack(nums, track);

        track.pop_back();
        used[i] = false;
    }
}
```

考虑一个例子：`nums = {1, 2, 2'}`，如果按照标准的全排列来写，那么会产生的结果是：
```
[
    [1,2,2'],[1,2',2],
    [2,1,2'],[2,2',1],
    [2',1,2],[2',2,1]
]
```
我们只要其中的
```
[ [1,2,2'],[2,1,2'],[2,2',1] ]
```
观察一下，这里的重复元素2，都是2在前，2'在后。这样一想：如果我们保证**相同元素在排列中相对位置保持不变**，通过这个限定剪枝，就能够得到正确结果。用代码语言就是：**前面的元素如果和我相同，那么他没有没选，我不能被选**。这个剪枝长成这样：
```C++
// 剪枝逻辑，固定相同的元素在排列中的相对位置
if (i > 0 && used[i - 1] == used[i] && !used[i - 1]) {
    // 如果前面相邻的元素和我相同，而且没有被用过，我不能被选！
    // i > 0 是为了防止触及 used[-1]
    continue;
}
```
因此，这种写法是：
```C++
vector<vector<int>> res;
vector<int> track;
vector<bool> used;

vector<vector<int>> permuteUnique(vector<int>& nums) {
    // 先排序，让相同的元素靠在一起
    sort(nums.begin(), nums.end());
    used = vector<bool>(nums.size(), false);
    backtrack(nums);
    return res;
}

void backtrack(vector<int>& nums) {
    if (track.size() == nums.size()) {
        res.push_back(track);
        return;
    }

    for (int i = 0; i < nums.size(); i++) {
        // 排列正常剪枝
        if (used[i]) {
            continue;
        }
        // 保持元素之间相对顺序不变
        // 如果前一个树枝和我是相同的数，但是他没有被用到 -> 我不能被选择！
        if (i > 0 && nums[i - 1] == nums[i] && !used[i - 1]) {
            continue;
        }

        // 做选择
        track.push_back(nums[i]); used[i] = true;
        // 进入下一个节点
        backtrack(nums);
        // 撤销选择
        track.pop_back(); used[i] = false;
    }
}
```

## 子集 / 组合（元素无重复，可以重复选）
### (LeetCode39.组合总和)[https://leetcode.cn/problems/combination-sum/]
我们通过`start`保证了元素顺序相对不变，前面的元素一定在后面元素的前面。

这个过程防止重复答案的出现，这里也需要。如果元素可以重复取，那我们进入下一层节点时，让下一层也可以用上一层“刚选择”的元素即可。也就是
```C++
backtrack(nums, i);
```
同时，我们需要合适的返回时机，多一个`trackSum`来记录路径和，当达到这个值之后直接返回即可。
综上，翻译成代码语言如下：
```C++
vector<vector<int>> res;
// 记录回溯的路径
deque<int> track;
// 记录 track 中的路径和
int trackSum = 0;

vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
    backtrack(candidates, 0, target);
    return res;
}

void trackback(vector<int> nums, int start, int target) {
    if (trackSum == target) {
        res.push_back(track);
        return;
    }
    // 如果超过目标和，停止向下遍历，否则会无限向下
    if (track > target) {
        return;
    }

    // 在这里保证元素顺序
    for (int i = start; i < nums.size(); i++) {
        // 做选择
        track.push_back(nums[i]); trackSum += nums[i];
        // 递归下一层回溯树，下一层还可以选 nums[i]
        backtrack(nums, i, target);
        // 撤销选择
        track.pop_back(); trackSum -= nums[i];
    }
}
```

## 排列（元素无重复，可以重复选）
实际上考虑排列树，每一次在 `n` 个元素中任意选一个即可，不需要 `used[]` 数组，让排列树树高 `== nums.size()`，返回即可：
```C++
vector<vector<int>> res;
deque<int> track;

vector<vector<int>> permuteRepeat(vector<int>& nums) {
    backtrack(nums);
    return res;
}

void backtrack(vector<int>& nums) {
    if (track.size() == nums.size()) {
        res.push_back(vector<int>(deque.begin(), deque.end()));
        return;
    }
    // 回溯算法标准框架，遍历子树
    for (int i = 0; i < nums.size(); i++) {
        // 做选择
        track.push_back(nums[i]);
        // 进入下一层回溯树
        backtrack(nums);
        // 取消选择
        track.pop_back();
    }
}
```
