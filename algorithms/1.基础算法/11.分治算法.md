# 分治算法框架
分治的思想是广泛存在的，但需要注意的是：只有部分问题可以通过分治提高效率。
> 假设有某问题规模 $N = a + b$，使用$O(N^2)$的算法求解，和分治后使用$O(N^2)$求解，有$O((a+b))^2 < O(a^2+b^2)$

**分治思想**和**分治算法**这两个概念不太一样，如下。
## 分治思想
分治思想无处不在，在二叉树问题中的**分解问题的思路**就是再说这件事。
分治思想就是将问题分解成若干个子问题，然后分别解决这些子问题，最后合并子问题的解得到原问题的解。广泛存在于**递归**算法中。
- 动态规划
- 后续遍历
- 分解思想 

这三个大部分时候是一回事。经常用递归的方法（树遍历）去求解。

## 分治算法
分治算法也是用分治思想，但有一个特点：**将问题分解后，相比不分治直接求解，时间复杂度更低**。经典的分治算法：
- 桶排序：找到最大最小值 -> 通过除法，将元素根据大小关系分入多个桶中 -> 对每个桶排序 -> 合并桶

分治算法有如下特征：
1. 分治思想是广泛存在的，几乎所有算法都可以写成递归分治的形式；
2. 分治思想不等于高效。很多时候将算法改成分治，凭空带来多余空间，但没有实际时间上的优化；
3. 用**二分的方式**分治，可以将递归树的深度从 $O(n)$ 降低到 $O(\log n)$，确实有优化效果。

### 例
如[LeetCode23.合并K个升序链表](https://leetcode.cn/problems/merge-k-sorted-lists/)。如果不使用经典的堆解法，考虑最基础的想法：不断合并两个有序链表，最终能够获得有序链表。
```C++
ListNode* mergeKLists(vector<ListNode*>& lists) {
    // 不断合并两个链表
    // 具体实现是将 k 个有序链表都合并到 第一个链表lists[0] 上
    if (lists.empty()) {
        return nullptr;
    }

    ListNode* l = lists[0];
    for (int i = 1; i < lists.size(); i++) {
        l = mergeTwoLists(l0, lists[i]);
    }

    return l;
}
```
其中，`mergeTwoLists`是合并两个有序链表，通过双指针实现。假设链表的长度为: ${l1, l2, ..., lk}$，那么在这 k 次合并，时间复杂度是：$O(l1 + l2) + O(l1 + l2 + l3) + ... + O(l1 + l2 + ... + lk)$。可见：**越靠前的链表被重复遍历的次数越多，这是这个算法低效的原因**。

为了尝试优化这个算法，将其改成递归形式，看看其递归树长什么样。
```C++
ListNode* mergeKLists(vector<ListNode*>& lists, int start) {
    if (start == lists.size() - 1) {
        return lists[start];
    }

    ListNode* subProblem = mergeKLists(lists, start + 1);
    
    // 将 lists[start] 和 后面的所有节点合并，就是原问题的解
    return mergeTwoLists(lists[start], subProblem);
}
```
这棵树退化成了单链表，越靠前的链表被遍历的次数越多。通过二分降低树的高度，如下：
```C++
ListNode* mergeKLists(vector<ListNode*>& lists, int start, int end) {
    if (start >= end) {
        return lists[start]; // #TODO
    }

    int mid = (left + right) / 2;
    ListNode* leftList = mergeKLists(lists, left, mid);
```
