# 回溯算法
回溯算法重点关注的是树枝，每一个叶子节点都放着一个合法答案。
因此：解决一个回溯问题，实际上就是一个遍历决策树的过程，树的每个节点都放着一个合法答案，将决策树遍历一边，将叶子节点上的答案都收集起来，就能得到合法答案。

回溯算法需要重点思考 3 个问题，站在一棵回溯树上：
1. 路径：已经做出了哪些选择？
2. 选择列表：当前还可以做出的选择？
3. 结束条件：到达决策树底层，无法再做选择的条件是什么？

伪代码框架如下
```python
result = []
def backtrack(路径, 选择列表):
    if 满足结束条件:
        result.add(路径)
        return
    
    for 选择 in 选择列表:
        做选择
        backtrack(路径, 选择列表)
        撤销选择
```
需要重点关注的是：
- 在递归调用前做选择，递归调用后撤销选择；
- 做选择、撤销选择都是在for循环内，只有在 for 循环内能够同时拿到初始节点和中点这两个节点。
- 属于遍历问题，因此函数一般没有返回值，用一个数据结构记录路径。
- 做选择、撤销选择的位置并不是多叉树“经典”前序遍历、后序遍历的位置。这点没错！但是这并不影响他是“进入一个节点”做选择 + “退出一个节点”撤销选择的过程。

为了能够清楚记录每个节点是否 “在路径上” 和 “是否被选择”，一般需要两个数据结构来记录：
1. 选择列表，如 `used`；
2. 路径，如 `track`。
综上，上面框架 for 循环更详细来说，应该如下：
```python
for 选择 in 选择列表:
    # 做选择
    将该选择（节点）从选择列表中移除
    路径.add(选择)
    backtrack(路径, 选择列表)
    # 撤销选择
    路径.remove(选择)
    将该选择再加入选择列表
```

### 全排列问题
```C++
#include <vector>
#include <list>

class Solution {
    vector<vector<int>> res;
public:
    // 主函数，输入一组不重复的数字，返回它们的全排列
    vector<vector<int>> permute(vector<int>& nums) {
        // 记录「路径」
        list<int> track;
        // 「路径」中的元素会被标记为 true，避免重复使用
        vector<bool> used(nums.size(), false);

        backtrack(nums, track, used);
        return res;
    }
private：
    // 路径：记录在 track 中
    // 选择列表：nums 中不存在于 track 的那些元素（used[i] 为 false）
    // 结束条件：nums 中的元素全都在 track 中出现
    void backtrack(const vector<int>& nums, list<int>& track, vector<bool>& used) {
        // 触发结束条件
        if (track.size() == nums.size()) {
            res.push_back(vector<int>(track.begin(), track.end()));
            return;
        }

        for (int i = 0; i < nums.size(); i++) {
            // 排除不合法的选择
            if (used[i]) {
                // nums[i] 已经在 track 中，跳过
                continue;
            }
            // 做选择
            track.push_back(nums[i]);
            used[i] = true;
            // 进入下一层决策树
            backtrack(nums, track, used);
            // 取消选择
            track.pop_back();
            used[i] = false;
        }
    }
};
```
