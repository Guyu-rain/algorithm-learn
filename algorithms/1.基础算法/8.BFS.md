# BFS 算法框架
BFS 算法关注每个节点，本质上还是将解空间建立起来然后去遍历。本质上还是树 / 图的遍历，如下是一个**记录步数**的**图 BFS 遍历**模板：
```C++
// 从 s 开始 BFS 遍历图的所有节点，并记录遍历的步数
// 当走到目标节点 target 时，返回步数
int bfs(const Graph& graph, int s, int target) {
    // 两个关键数据结构
    // 防止成环（死循环）的visited
    vector<bool> visited(graph.size(), false);
    // BFS 需要一个队列
    queue<int> q;

    q.push(s);
    visited[s] = true;
    
    // 记录从 s 开始走到当前节点的步数
    int step = 0;
    while (!q.empty()) {
        int sz = q.size();
        for (int i = 0; i < sz; i++) {
            int cur = q.front(); q.pop();
            cout << "visit " << cur << " at step " << step << endl;
            // 判断是否到达中点
            if (cur == target) {
                return step;
            }
            for (int to : neighborsOf(cur)) {
                if (!visited[to]) {
                    q.push(to);
                    visited[to] = true;
                }
            }
        }
        step++;
    }
    return -1; // 到达不了 target
}
```

其中，`onPath / visited`数组如果键是字符串或者`unhashable`的类型（没法开`bool`数组），可以使用hashSet，效率也很高。
需要格外注意的是：当有一个新元素要入队之前，及时将其加入`onPath / visited`数组，避免死循环！

## 双向 BFS 优化
如果一般的 BFS 超时，而且已知起点和终点，那可以考虑一下双向 BFS。
双向 BFS 同时从起点和终点扩散、直到两者有交集时停止。

以[LeetCode.752](https://leetcode.cn/problems/open-the-lock/)为例，注意以下算法细节。
```C++
int openLock(vector<string>& deadends, string target) {
    unordered_map<string> deads(deadends.begin(), deadends.end());

    // base case
    if (deads.count("0000")) return -1;
    if (target == "0000") return 0;

    // 使用集合不用队列，可以快速判断元素是否存在
    unordered_set<string> q1, q2;
    unordered_set<string> vistited;

    int step = 0;
    q1.insert("0000");
    visited.insert("0000");
    q2.insert(target);
    visited.insert(target);

    while (!q1.empty() && !q2.empty()) {
        // 在这里增加步数
        step++;

        // 注意：hashSet 在遍历时不可以修改！如果修改可能出现重复遍历
        // 用一个新的 hashSet newQ1 来存储邻居节点
        unordered_map<string> newQ1;
        
        // 获取 q1 所有的邻居节点
        for (const string& cur : q1) {
            // 将没有遍历过的邻居节点加入集合
            for (const string& neighbor : getNeighbors(cur)) {
                // 判断是否到达终点
                if (q2.count(neighbor)) {
                    return step;
                }
                // 某个节点没有访问，且不是死节点
                if (!visited.count(neighbor) && !deads.count(neighbor)) {
                    visited.insert(neighbor);
                }
            }
        }

        // newQ1 存储 q1 的邻居节点
        q1 = newQ1;
        // 每次 BFS 都扩散 q1，选择 q1 和 q2 中较小的一个来进行扩散
        // 实际上就是“同时”操作两头，用一个 step 来维护两边走的步数
        // 每次扩散小的可以减少复杂度
        if (q1.size() > q2.size()) {
            std::swap(q1, q2);
        }
    }
    return -1;
}

// 几个工具函数
string plusOne(string s, int j) {
    if (s[j] == '9')
        s[j] = '0';
    else
        s[j] += 1;
    return s;
}

// 将 s[i] 向下拨动一次
string minusOne(string s, int j) {
    if (s[j] == '0')
        s[j] = '9';
    else
        s[j] -= 1;
    return s;
}

vector<string> getNeighbors(string s) {
    vector<string> neighbors;
    for (int i = 0; i < 4; i++) {
        neighbors.push_back(plusOne(s, i));
        neighbors.push_back(minusOne(s, i));
    }
    return neighbors;
}
```
需要注意的点：
1. 不再使用队列存储元素，而是改用 hashSet，方便快速判断两个集合是否有交集；
2. 注意：调整了 `return step` 和 `step++` 的位置。当 `q1` 中的元素出现在了 `q2 `时，需要return，此时这一步没有“迈出去”，因此需要提前++；
3. 每次保持 `q1` 是元素较少的集合，这样可以一定程度减少搜索次数。

在理论上，BFS 和双向 BFS 都是 O(N) 的时间复杂度，都是相同的，在实际中双向会快一点。
