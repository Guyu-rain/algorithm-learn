# 滑动窗口
滑动窗口本质上还是**双指针（快慢指针）**，滑动窗口主要解决的问题是**子串 / 子数组**。

对于子数组问题，可以通过遍历起点与终点(`0 <= start <= end <= nums.size()`)的所有可能性来遍历，最简单的想法如下, 时间复杂度显然是 $O(n^2)$
```C++
for (int i = 0; i < nums.size(); i++) {
    for (int j = i; i < nums.size(); j++) {
        // nums[i..j] 是一个子数组
    }
}
```

可以通过不让内层循环回退（一直向后遍历）的方式，让时间复杂度下降到$O(N)$。
```C++
int left = 0, right = 0;

// 索引区间 [left, right) 是一个窗口
while (right < nums.size()) {
    // 增大窗口
    window.addLast(nums[right]);
    right++;
    while (window needs shrink) {
        // 缩小窗口，内层循环 left 不回退
        window.removeFirst(nums[left]);
        left++;
    }
}
```

> 请注意：滑动窗口本质上也是对暴搜的优化，用已知信息（串/数组特点）进行优化；滑动窗口并不穷举所有子串，而是规避冗余、找到可能解集进行遍历；
```C++
// 滑动窗口框架
void slidingWindow(string s) {
    // 用合适的数据结构记录窗口中的数据
    // 如果需要记录窗口中元素出现的次数，用 map
    // 如果需要记录窗口中元素和，可以只用一个 int
    auto window = ...

    int left = 0, right = 0;
    while (right < s.size()) {
        // c 是即将移入窗口的元素
        char c = s[right];
        window.add(c);
        // 增大窗口
        right++;

        // 进行窗口数据的一系列更新
        ... 
        
        // *** debug(调试) 输出的位置 ***
        printf("window: [%d, %d)\n", left, right);

        // 判断做窗口是否要收缩
        while (window needs shrink) {
            // d 是即将移出窗口的字符
            char d = s[left];
            window.remove(d);
            // 缩小窗口
            left++;

            // 进行窗口内的一系列更新
            ... 
        }
    }
}
```
上述框架是两个循环嵌套，不断维护窗口`window`。两个指针都不回退。
框架两处`...`表示需要更新窗口数据的地方，需要填入具体的处理逻辑。两处操作分别是扩大、减小窗口的更新操作，是对称的。

每个移动窗口问题，回答三个问题：
1. 什么时候应该移动`right`扩大窗口？窗口加入字符时，应该更新哪些数据？
2. 什么时候窗口应该暂停扩大，开始移动`left`缩小窗口？从窗口移出字符时，应该更新哪些数据？
3. 什么时候应该更新结果？

## 例题
### 寻找最小子串
#### 滑动窗口思路：
1. 初始化`left = right = 0`，将索引**左闭右开**区间`[left, right)`称为一个“窗口”。
> “左闭右开”的原因是：`left = right`时，此时窗口没有元素，能够自然包含空串这种情况。
> 需要注意，窗口的大小是`right - left`
2. 不断增加`right`指针，扩大`[left, right)`，直到窗口中的字符串符合要求（包含了`T`中的所有字符）
3. 此时，停止增加`right`，转而不断曾姐`left`指针缩小窗口`[left, rigth)`，直到窗口中的字符串不再符合要求（不包含`T`中的所有字符）。同时，每次增加`left`，会丢失上一次的结果，需要更新一下结果。
4. 重复`2, 3`，直到`right == S.size()`。
```C++
string minWindow(string s, string t) {
    unordered_map<char, int> need, window; // 或者只用一个 need 来记录
    // 初始化 need
    for (char c : t) {
        need[c]++;
    }
    // 维护一个窗口
    int left = 0, right = 0;
    // 记录 window 中的字符满足 need 条件的字符个数
    int valid = 0;
    // 记录最小覆盖子串的起始索引及长度 —— 记录结果
    int start = 0, len = INT_MAX; 
    while (right < s.size()) {
        char c = s[right];
        right++;
        // 进行窗口的一系列更新
        if (need.count(c)) {
            window[c]++;
            if (window[c] == need[c]) {
                valid++; // 有一个字符已经满足了要求
            }
        }

        // 判断是否需要收缩
        while (valid == need.size()) {
            // 更新最小覆盖子串 —— 记录结果
            if (right - left < len) {
                start = left;
                len = right - left;
            }
            // d 是即将移出窗口的字符
            char d = s[left];
            left++;
            // 进行窗口的一系列更新
            if (need.count(d)) {
                if (window[d] == need[d]) {
                    valid--;
                }
                window[d]--;
            }
        }
    }
    return len == INT_MAX ? "" : s.substr(start, len);
}
```
