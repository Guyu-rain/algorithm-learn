# 双指针技巧
当`链表`本身有序、或者有多个`链表`时，就应该考虑双指针技巧，这里的顺序表包括。

对于链表的处理，经常使用 `dummy`虚拟头节点/哨兵节点 技巧，用来占一个“空位置”，避免处理空指针的情况。
换言之，一般情况应该定义具有虚拟头节点/哨兵节点的链表。

## 例题
### 双指针解决链表相关问题
#### 单链表的倒数第 `k` 个节点
一个有`n`个节点的单链表，倒数第`k`个节点 等价于 正数`n-k+1`个节点，如果是索引（从0开始），则是`[n-k]`。

`n`未知，遍历一次返回倒数第 `k` 个节点，我们需要知道如何向前走`n-k`个节点：
1. 指针`p1`先向前走`k`个位置，此时`p1`还要走`n-k`个位置，会遍历到`nullptr`(链表尾)；
2. 此时，用一个指针`p2`，和`p1`一起向前走，当`p1`到了`nullptr`时，`p2`也就到了`n-k+1`的位置。

> 许多单链表都需要获取倒数某个节点的信息，这是一个只遍历一次就能拿到该节点的模板方法。
```C++
ListNode* k_thFromEnd(ListNode* head, int k) {
    ListNode *p1 = head, *p2 = head;
    while(k--) {
        p1 = p1->next;
    }

    while (p1 != nullptr) {
        p1 = p1->next;
        p2 = p2->next;
    }

    return p2;
}
```

#### 链表的中点
设链表的长度为`N`，用两个指针 `fast`, `slow`，`slow`每次走一步，`fast`每次走两步；如此`fast`到达终点时，`slow`到达中点；
```C++
LinkNode* middleNode(LinkList* head) {
    LinkNode* slow = head, *fast = head;
    while (fast->next != nullptr && fast != nullptr) {
        fast = fast->next->next;
        slow = slow->next;
    }

    return slow;
}
```
> 如果链表有偶数个节点(如只有一个节点)，那么会`slow`最后会停在`N/2`，也就是会返回靠后的一个节点。

#### 判断链表是否包含环
使用快慢指针`slow`, `fast`，如果`fast`指针能够正常到达末尾节点，链表没有环；如果`slow`, `fast`能够相遇，那么一定是有环
```C++
bool hasCycle(ListNode *head) {
    ListNode *slow = head, *fast = head;
    while (fast != nullptr && fast->next != nullptr) {
        show = show->next;
        fast = fast->next->next;
        // 两指针相遇，说明有环
        if (slow == fast) {
            return true;
        }
    }
    // 没有环
    return false;
}
```

##### 计算环的起点
使用快慢指针`slow`，`fast`。两者相遇时，假设`slow`走了`k`步，那么`fast`指针一定走了`2k`步。多出来的`2k - k = k`一定是在环里面打转，因此`k`一定是环的正数倍。
此时，假设`slow`指针进入环之后走了`m`步与`fast`相遇，那么从`head`开始走`k-m`步，可以到环的起点。而`fast`指针此时再走`k-m`步，刚好可以到环起点。
所以有如下算法：
1. 用`slow`, `fast`两个指针直到相遇；
2. 相遇时，将`slow`指针换到`head`
3. `slow`，`fast`两个指针都以步长为`1`继续前进，直到两者相遇，即为环的起点
> 可以画一个图辅助理解
```C++
ListNode *detectCycle(ListNode *head) {
    ListNode *fast, *slow;
    fast = slow = head;
    while (fast != nullptr && fast->next != nullptr) {
        fast = fast->next->next;
        slow = slow->next;
        // 两者相遇
        if (fast == slow) break;
    }
    if (fast == nullptr || fast->next == nullptr) {
        // 如果是因为到了结尾，说明没有环
        return nullptr;
    }

    // 重新指向头节点
    slow = head;
    while (slow != fast) {
        slow = slow->next;
        fast = fast->next;
    }
    return slow;
}
```

#### 计算两个链表是否相交
假设两个链表包含不相交区域$A$，$B$ 和相交区域$C$
$$ 
链表A：A \to C \\
链表B：B \to C
$$
最自然的想法，是遍历第一个链表，并用`hashSet`记录都有哪些节点在第一条链表上；然后再遍历第二条链表，如果遇到相同节点(`hashSet.contains(p)`)，那么就是相交；

如果不想有额外的空间开销，应该解决的问题，是：
```
如何让两个指针能够同时进入相交的区域
```
解决方案：`p1`遍历完A链表后，去遍历B链表；`p2`遍历完B链表后，遍历A链表。这样，再进入相同区域$C$之前，两者经过的长度都是$|A| + |B| + |C|$，能够同时进入相交区域。
```C++
ListNode* getIntersectionNode(ListNode* headA, ListNode* headB) {
    ListNode* p1 = headA, *p2 = headB;
    while (p1 != p2) {
        // p1 是否已经走到结尾? 如果已经到结尾，去headB
        p1 = (p1 == nullptr) ? headB : p1->next; 
        // p2 是否已经走到结尾? 如果已经到结尾，去headA
        p2 = (p2 == nullptr) ? headA : p2->next;
    }
    // 注意：如果没有公共节点，那么p1, p2会遍历完完整A, B两条链表同时到达 nullptr
    // 可以正确返回 nullptr
    return p1;
} 
```
