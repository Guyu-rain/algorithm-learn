# 贪心算法框架
## 贪心选择性质
贪心选择性质：**能够通过局部最优解直接推导出全局最优解**。
> 举一个例子：现有面额 1¥ 和 100¥ 的钞票，都是无限量。
> 问题1：选择 10 张，让总金额最大；
> 问题2：给定一个金额 amount，用尽量少的钞票数量达到。
> 问题1 具有贪心选择性质，因为每次选择 100¥ 的钞票（局部最优）可以得出全局最优解；而 问题2，如果 amount = 99，显而易见：局部最优解是选择 100¥，但在这里做不到。只能选择 99张 1¥，因此不具有最优选择性质。
```C++
// 问题 1：分解（关注子树） + 贪心选择性质剪枝
int maxAmount(int n) {
    // base case
    if (n == 0) {
        return 0;
    }

    // 因为有最优子结构性质，因此每次直接返回 100 即可
    // 本质上是后续遍历，关注子树
    return maxAmount(n - 1) + 100;
}

// 问题 2：分解（关注子树）
int minCount(int amount) {
    // base case
    if (amount == 0) {
        return 0;
    }
    if (amount < 0) {
        return -1; // 无解
    }

    int leftMin = (minCount(amount - 1) == -1) ? -1 : minCount(amount - 1);
    int rightMin = (minCount(amount - 100) == -1) ? -1 : minCount(amount - 100);

    if (leftMin == -1) {
        if (rightMin == -1) {
            return -1;
        }
        return rightMin;
    }
    return leftMin;
}
```

### 贪心选择性质 & 最优子结构
- 最优子结构（动态规划）：“我已经将所有子问题的最优解都解出来了，然后我可以基于这些子问题的最优解推出原问题的最优解，子问题的答案不会影响原问题”。
- 贪心选择性质（贪心）：“我只需要进行一些局部最的选择策略，就直接能够知道哪个子问题的解是最优的，而且这个局部最优解可以推导出原问题的最优解，我只需要不停执行这个策略就能获得原问题的最优解。我此时此刻就知道这个策略是什么”。

## 例
### [LeetCode45.跳跃游戏II](https://leetcode.cn/problems/jump-game-ii/)
首先，拿到这个题先想穷举，从第一个节点开始，有`1..nums[0]`这一共`nums[0]`中选择。我们需要选择这些子问题（子树）的最小者，因此，这个题目自然适合：`动态规划 / 分解方式 / 后续遍历`。记从节点`n`跳到最后一个位置`nums.size()-1`最少需要$dp(n)$步：
$$
dp(n) = 
\begin{cases}
0 &, n = 0 \\
min\{dp(n + i) + 1 | i\in[1, nums[n]]\} &, else
\end{cases}
$$
用带有 `memo` 的动态规划实现一下：
```C++
// 假设 n 是一个 const int
// 用于记录是否可以到达最后，用于求最小，因此初始化成 INT_MAX，这里用不可能到达的大值 n 来代替
vector<int> memo(n, n); 
vector<int> nums; // 题目输入

// 记从下标 p 到最后一个位置至少需要 dp(p) 步
int dp(int p) {
    // base case
    if (p >= nums.size() - 1) {
        return 0;
    }
    if (memo[p] != n) {
        return memo[p];
    }

    // 遍历子问题（子树）
    for (int i = 1; i <= nums[p]; i++) {
        // 穷举所有选择：如果出现了 p + i >= nums.size()，说明一步就能够达到，属于 base case
        // 因此 dp(p + i) 能够获得正确的结果
        int subProb = dp(p + i);
        memo[p] = min(subProb + 1, memo[p]);
    }
    
    return memo[p];
}
```

但是动态规划会超时，加入`memo`后，时间复杂度是$O(n^2)$。一般穷举 + 备忘录还是超时，就该考虑有没有贪心选择性质。

仔细观察一个子问题，假设我们有如下的数组：
```C++
nums[] = {3, 1, 4, 2, ...}
```
一开始，我们在`idx = 0`，那么接下来可以去`idx = 1, 2, 3`。我们尝试进行贪心策略：加下来会在这几个位置选择哪个呢？
为了能走的更远，我们应该选择`idx + nums[idx]`最大者，也就是`idx = 2`的位置。这个局部最优解显然能够获得全局最优解。我们需要不停确认边界，然后从头向后扫一次即可。将这个过程翻译成代码：
```C++
// 假设 N 是一个 const int
vector<int> memo(N, N); 
vector<int> nums; // 题目输入

int jump() {
    int n = nums.size();
    // base case, 如果没有元素 或 只有一个元素
    if (n <= 1) {
        return 0;
    }

    int curFarthest = 0;
    int end = 0, jumps = 0;
    for (int i = 0; i < n - 1; i++) {
        curFarthest = max(curFarthest, i + nums[i]);
        // 如果到达了 end，需要考察新的 end
        if (i == end) {
            // 走到这个区间中，能够到达最远的点
            jumps++;
            end = curFarthest;
            // 然后判断是否可以到达终点
            if (n - 1 <= end) {
                // 如果最远 cover 了终点：可以到达终点，直接返回
                return jumps;
            }
        }
    }
    // 到这里说明 end < i == (n - 1)，也就是最远也到不了终点
    return -1;
}
```
### 对于[leetCode55.跳跃问题](https://leetcode.cn/problems/jump-game/description/)
是判断是否能够到达终点，延续上面的思想，考察一个最小子问题：
```C++
nums = {2, 3, 1, 1, 4}
```
在`idx = 0`的位置，你可以选择跳跃到`idx = 1, 2`这两个位置，我们尝试贪心：选择`idx + nums[idx]` 最大者`idx = 1`。然后循环往复，可以找到这个问题的最优解。而本题只需要考察：每个**能够到达的位置，能够去到的最远位置`farthest`**，如果有某个位置`i` >= 当前能够到达的最远距离`farthest`（这个最远距离包含`i`这个位置能够到达的最远距离），说明到这里已经无法继续向后了，也就是不能到达终点。
> 这个不能到达的原因，是在这之前最远能到达`i`，而`nums[i]=0`。
翻译成代码语言如下：
```C++
vector<int> nums;

bool canJump() {
    int n = nums.size();
    // base case
    // 注意：[0] 应该输出 false，下面能够包含这种情况，因此不写 base case

    int farthest = 0;
    for (int i = 0; i < n - 1; i++) {
        farthest = std::max(farthest, i + nums[i]);
        if (i >= farthest) {
            return false;
        }
        // 遇到了 0，而且前面的 nums 导致到这里跨不过去
        if (farthest >= n - 1) {
            return true;
        }
    }
    // 如果能出循环，最后一次是 i = n - 2，i < farthest
    // 也就是 farthest 最近也是 n - 1
    // 所以其实这个判断没有必要，在这里只是保险
    return farthest >= (n - 1);
}

```

## 总结一下
一般题目很难看出来是否有**贪心选择性质**，像例题中，这个贪心选择性质并不是很容易能够想通并用代码表示出来。
一般考虑问题应该：
1. 穷举，能够写出正确搜索解空间的代码，如果超时：
2. 加入备忘录，看看是否能够不超时，如果还是超时：
3. 仔细考虑一个子问题，在不知道这个子问题的解时，看看能不能获得这个子问题的贪心解，然后重复这个过程，解决问题。
