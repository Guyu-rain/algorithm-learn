# 动态规划
- 动态规划的一般形式是**求最值**。
- 求解动态规划的核心是穷举。或者说，算法的核心就是穷举；
- 动态规划有三个核心要件：
    1. 状态**转移方程**，是描述状态转移的核心。动态规划就是要找出状态转移的方式，然后画出解空间树，通过自顶向下或者自底向上来解决。
    2. 问题具备**最优子结构**：能够通过子问题的最值求解原问题的最值；原问题分解出的子问题的最优化并不影响原问题的求解；
    3. 存在**重叠子问题**：需要通过 **备忘录** 或 **DP Table** 来优化穷举过程。

## 几个例子说明动态规划
### 斐波那契数列
斐波那契数列本质上是将大问题化成两个子问题，因此关注的是**子树**，解决方式关键是**后序遍历/分解思路/使用带有返回值的函数来进行递归**。
```C++
int fib(int n) {
    // base case
    if (n == 0 || n == 1) {
        return n;
    }

    int left = fib(n-1);
    int right = fib(n-2);
    return left + right;
}
```
一般见到的都是直接使用 `return fib(n-1) + fib(n-2);`，本质上就是后序遍历。
时间复杂度是 $O(n^2)$（考虑二叉树）。
#### 解决重叠子问题的重复计算
##### 带备忘录的递归解法
通过将已经计算出来的值存起来，如果这个值之前计算过，那么直接返回，这样避免了大量重复子问题的计算，时间复杂度是$O(n)$。
```C++
int fib(int n) {
    vector<int> memo(n + 1, -1);
    return dp(memo, n);
}

int dp(vector<int>& memo, int n) {
    // base case
    if (n == 0 || n == 1) {
        return n;
    }

    if (memo[n] != -1) {
        return memo[n];
    }
    // 先存入备忘录
    memo[n] = fib(memo, n - 1) + fib(memo, n - 2);
    return memo[n];
}
```

#### 自顶向下 & 自底向上
如果画出了动态规划的解空间（树），可以通过两种方式取遍历：
1. 自顶向下：带有`memo`备忘录的递归；
2. 自底向上：`DP table`，用 for 循环迭代`dp`数组进行求解；
##### `dp`数组的迭代（递推）解法
```C++
int fib(int n) {
    // 不需要计算
    if (n == 0 || n == 1) {
        return n;
    }

    vector<int> dp(n + 1);
    // base case
    dp[0] = 0, dp[1] = 1;
    // 状态转移
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-1];
    }

    return dp[n];
}
```
动态规划问题最困难的就是写出**状态转移方程**，也就是暴力解（穷举路径/解空间）
##### 对空间的优化
上面在计算`dp[i]`时，只用到了前两个状态，因此只需要记录`n-1`和`n-2`两个状态即可。这样用常数的额外空间，就能解出最后的结果。
```C++
int fib(int n) {
    if (n == 0 || n == 1) {
        return n;
    }

    // 分别代表之前的 dp[i-1], dp[i-2]
    // base case
    int dp_i_1 = 1, dp_i_2 = 0;
    for (int i = 2; i <= n; i++) {
        int dp_i = dp_i_1 + dp_i_2;
        // 滚动更新
        dp_i_2 = dp_i_1;
        dp_i_1 = dp_i;
    }
    return dp_i_1;
}
```

### 凑零钱
[LeetCode.332](https://leetcode.cn/problems/coin-change/description/)
如果用穷举的思路来理解这个过程，能够列出如下的状态转移方程：
$$
dp(amount) = 
\begin{cases}
0 &, amount = 0 \\
-1 &, amount < 0 (无解) \\
min\{dp(n-coin)+1|coin\in coins\} &, n>0 
\end{cases}
$$
需要特别注意的是，当$amount<0$时，这个状态是无效的，因此这条支路不能要
```C++
int coinChange(vector<int>& coins, int amount) {
    return dp(coins, amount);
}

// 定义：要凑出金额 n, 至少要 dp(coins, n) 个硬币
int dp(vector<int>& coins, int n) {
    // base case
    if (n == 0) {
        return 0;
    }
    if (n < 0) {
        return -1;
    }
    // 状态转移方程，是一个求 min 的标准流程
    int res = INT_MAX:
    for (int coin : coins) {
        // 子问题的解先拿到
        int subProb = dp(coins, n - coin);
        // 需要特殊处理无解的情况
        if (subProb == -1) {
            continue;
        }
        // 求 min 的标准过程
        res = std::min(res, subProb + 1);
    }

    return res == INT_MAX ? -1 : res;
}
```
- 通过备忘录来优化这个过程
```C++
const int N = 1e5 + 10;
// 注意：-1 也是一个合法返回的数，表示无解
vector<int> memo = vector<int>(N, -666); // 这个数永远不会被触及到

int coinChange(vector<int>& coins, int amount) {
    return dp(coins, amount);
}

// 定义：要凑出金额 n, 至少要 dp(coins, n) 个硬币
int dp(vector<int>& coins, int n) {
    // base case
    if (n == 0) {
        return 0;
    }
    if (n < 0) {
        return -1;
    }
    // 备忘录，防止重复计算
    if (memo[n] != -666) {
        return memo[n];
    }
    // 状态转移方程，是一个求 min 的标准流程
    int res = INT_MAX:
    for (int coin : coins) {
        // 子问题的解先拿到
        int subProb = dp(coins, n - coin);
        // 需要特殊处理无解的情况
        if (subProb == -1) {
            continue;
        }
        // 求 min 的标准过程
        res = std::min(res, subProb + 1);
    }

    memo[n] = (res == INT_MAX) ? -1 : res;;
    return memo[n];
}
```

- `dp` 数组的解法
```C++
int coinChange(vector<int>& coins, int amount) {
    // 定义 dp 数组, dp[0..amount]
    vector<int> dp(amount + 1, INT_MAX);

    dp[0] = 0;
    // 外层 for 循环遍历所有状态的可能值
    for (int i = 0; i < dp.size(); i++) {
        // 内层 for 循环遍历所有能到达 dp[i] 的可能，然后求最小值
        for (int coin : coins) {
            if (i - coin < 0) {
                continue;
            }
            dp[i] = min(dp[i - coin] + 1, dp[i]);
        }
    }
    return (dp[amount] == INT_MAX) ? -1 : dp[amount];
}
```
