# 滑动窗口
这里复习一下在`1.基础算法`中的`3.滑动窗口算法模板`。

窗口指的是`[left, right)`这样一个子数组。左闭右开的原因是当`left == right`时，窗口内没有元素，这是合法的。
```C++
void slidingWindow(string s) {
    // 使用合适的数据结构记录窗口中的数据，需要根据场景变通
    // 如果要记录窗口中元素出现的个数，使用 map
    // 如果要记录窗口的元素和，使用 int
    auto window = ...

    int left = 0, right = 0;
    while (right < s.size()) {
        // c 是即将移入窗口的字符
        char c = s[right];
        window.add(c);
        // 增大窗口
        right++;

        // 进行窗口内数据的一些列更新
        ...

        // *** debug，在这里看一下都是在什么时候开始缩小窗口的 ***
        printf("window: [%d, %d)\n", left, right);

        // 判断左侧窗口是否需要收缩
        while (window needs shrink) {
            // 💡 到这里是一个满足条件的窗口，经常在这里更新答案
            // d 是即将移除窗口的字符
            char d = s[left];
            window.remove(d);
            // 缩小窗口
            left++;

            // 进行窗口内数据的一系列更新
            ...
        }
    }
}
```
可以看出，窗口的扩张和收缩是非常对称的，遇到**子串 / 子数组**问题，应该想到滑动窗口，然后尝试回答：
1. 什么时候应该移动`right`扩大窗口？窗口加入字符时，应该更新那些数据？
2. 什么时候窗口应该暂停扩大，开始移动`left`缩小窗口？从窗口移除字符时，应该更新那些数据？
3. 什么时候更新结果？

只要能回答以上这三个问题，就说明可以用滑动窗口解决这个问题。这时候，需要找到一个合适的数据结构去描述**窗口中的信息和结果（答案）**，然后套公式，正确填入**窗口内数据更新的逻辑**即可。
> 注意：一般需要考虑记录：
> 1. 描述窗口的数据结构；
> 2. 描述答案的数据结构。

对模板需要说明：
- “3.什么时候更新结果”这个问题的答案：一般是在合法窗口（内层循环一进入）的地方进行更新。但也有可能在其他地方！因此要按题目随机应变。
- 同时，模板中`window.add(c)`, `window.remove(d)`不一定是必须的；只进行下面对窗口内数据的更新就足够了。

## 例题
### [LeetCode1658.将 x 减到 0 的最小操作数](https://leetcode.cn/problems/minimum-operations-to-reduce-x-to-zero/)
这个题目有两个要点：
1. **转换成子数组问题**：如果直接穷举，时间复杂度是$O(2^n)$，肯定会超时。这个题目看似是“两端”而不是“子数组”，我们只需要一个转换就能将其转换成“中间的滑动窗口问题”：求最大子窗口，满足**其中元素之和` = sum(nums) - x`**。
2. **什么时候收缩窗口，什么时候更新答案**：显而易见，当窗口内元素之和` == sum(nums) - x`时尝试更新答案，当窗口内元素之和`> sum(nums) - x`时缩小窗口；
```C++
int minOperations(vector<int>& nums, int x) {
    int n = nums.size(), sum = 0;
    for (int i = 0; i < n; i++) {
        sum += nums[i];
    }

    int target = sum - x;

    int left = 0, right = 0;
    int windowSum = 0;
    int maxLen = INT_MIN;
    while (right < nums.size()) {
        // 扩大窗口
        windowSum += nums[right];
        right++;

        // 注意：如果 windowSum 一直 > target，left 会越界
        while (windowSum > target && left < right) {
            // 缩小窗口
            windowSum -= nums[left];
            left++;
        }
        // 注意在这里更新答案：能够覆盖所有情况（包括空串）
        if (windowSum == target) {
            maxLen = max(maxLen, right - left);
        }
    }
    return maxLen == INT_MIN ? -1 : n - maxLen;
}
```

### [LeetCode424.替换后最长的重复字符](https://leetcode.cn/problems/longest-repeating-character-replacement/)
与上一题实际上相同，只不过不是`2`类，而是所有的大写字母。我们每次需要将最多的字母看成一类，其他的看成一类，然后进行更新。

记录 window 得数据结构很值得学习：如何记录字母。思路总体与上题相同，如下：
```C++
int characterReplacement(string s, int k) {
    int left = 0, right = 0;
    // 统计窗口中每个字符出现的次数
    int windowCharCount[26] = {0}; // 开一个 26 大小的数组，注意：C 原生数据类型必须初始化！！！
    // 记录窗口中字符的最多重复次数
    int windowMaxCount = 0;
    // 结果的记录
    int res = 0;

    while (right < s.length()) {
        // 扩大窗口
        int c = s[right] - 'A';
        right++;

        // 处理窗口
        windowCharCount[c]++;
        windowMaxCount = max(windowMaxCount, windowCharCount[c]);

        // 当窗口不符合题意时缩小窗口：
        // window.size - windowMaxCount > k 时，窗口需要替换的元素必然 > k，必须缩小窗口
        // 需要注意的是，这里 left++ 之后，循环一定会被打破，所以这里等价于写 if
        while (right - left - windowMaxCount > k) {
            // 右移
            char d = s[left];
            left++;

            // 更新窗口
            windowCharCount[d - 'A']--;
            // 这时窗口中 windowMaxCount 不会改变
            // 最坏的情况是有字符数目相同，而且其中有字符在这里 -1，此时最大还是 windowMaxCount
            // 总之，windowMaxCount 不会改变
        }
        // 这里一定是一个合法的窗口
        res = max(res, right - left);
    }
    return res;
}
```

### [LeetCode219.存在重复元素 II](https://leetcode.cn/problems/contains-duplicate-ii/)
还是线性数据结构 + 子数组（虽然不是明说，但是有边界，很明显有子数组的感觉）。考虑滑动窗口。注意⚠️这里有一个很明显的可行性约束：子数组大小必须小于等于 k：
1. 什么时候`right++`？当窗口大小小于 k 时。
2. 什么时候`left++`？当窗口大小大于 k 时；
3. 什么时候更新答案？当窗口元素等于 k，且发现窗口内存在重复元素时。
```C++
bool containsNearbyDuplicate(vector<int>& nums, int k) {
    int left = 0, right = 0;
    unordered_set<int> window;
    while (right < nums.size()) {
        // 在这里，窗口大小一定小于 k（因为这里已经经过收缩，为合法大小）
        if (window.find(nums[right]) != window.end()) {
            return true;
        }
        window.insert(nums[right]);
        right++;

        if (right - left > k) {
            window.erase(nums[left]);
            left++;
        }
    }
    return false;
}
```

### [LeetCode220.存在重复元素 III](https://leetcode.cn/problems/contains-duplicate-iii/)
和上一题很相似，滑动窗口帮我们限制住了 `indexDiff`，我们需要自己处理 `valueDiff`。需要能够自由索引 + 排序，需要用到二叉搜索树`set`。

二叉搜索树`set`有求：第一个**大于等于** `val` 的元素的方法`set.lower_bound(val)`以及第一个**小于**`val`元素的方法`set.upper_bound(val)`。这刚好能够满足求“最小差距”的要求（不用全部比较）。

每加入一个元素之前，先判断是否有满足题意的元素。如下：
```C++
#include <set>
#include <vector>

bool containsNearbyAlmostDuplicate(vector<int>& nums, int indexDiff, int valueDiff) {
    set<long> window;
    int left = 0, right = 0;
    while (right < nums.size()) {
        // 题目要求 i!=j，我们先进行判断，然后再扩大窗口
        // 查找大于等于 nums[right] 的最小元素
        auto ceiling = window.lower_bound(nums[right]);
        // 判断是否成立
        if (ceiling != window.end() && *ceiling - nums[right] < valueDiff) {
            return true;
        }
        // 查找小于 nums[right] 的元素
        // --ceiling 就是 floor，但是要防止直接操作越界
        auto floor = ceiling;
        if (floor != window.begin()) {
            --floor;
            if (nums[right] - *floor <= valueDiff) {
                return true;
            }
        }
        window.insert(nums[right]);
        right++;
        // 索引不满足要求
        if (right - left > indexDiff) {
            window.erase(nums[left]);
            left++;
        }
    }
    return false;
}
```

### [LeetCode395.至少有K个重复字符的最长子串](https://leetcode.cn/problems/longest-substring-with-at-least-k-repeating-characters/)
还是子串问题。先尝试回答一下滑动窗口的三个问题：
1. 何时增加窗口？
2. 何时缩减窗口？
3. 何时记录答案？

但是发现，如果想尽可能装入更多的字符，需要一直扩张窗口，没有收缩的时机！

理论上来说，这个题并不适用滑动窗口，因为找不到合适的时机进行收缩。但我们尝试加入一些约束：我们求每个字符都出现至少`k`次，而且仅包含`count`种不同字符的最长子串。

这时候收缩的时机就很明确了。当串中已经满足了 `count` 种 至少出现过 `k` 次的字符时，就应该进行收缩。
1. 什么时候扩大窗口？窗口中字符种类小于`count`时扩大窗口；
2. 什么时候缩小窗口？窗口中字符种类大于`count`时扩大窗口；
3. 什么时候得到一个合法答案？窗口中所有字符都已经大于等于`k`时，子串为答案。

注意：我们这里因为增添了 `count` 种字符，所以收缩、扩张、更新答案的时机都很明确，收缩扩张都是依赖于这个`count`来进行的。然后在窗口中`元素种类 == count`时尝试更新答案。

而在外层，只需要遍历 26 种可能即可，$O(26N) = O(N)$，也不失为一种解法：
```C++
int longestSubstring(string s, int k) {
    int len = 0;
    for (int i = 1; i <= 26; i++) {
        // 限制窗口中只能有 i 种不同字符
        len = max(len, logestKLetterSubstr(s, k, i));
    }
    return len;
}

// 寻找 s 中含有 count 种字符，且每种字符出现次数都大于 k 的子串
int logestKLetterSubstr(string s, int k, int count) {
    int res = 0;
    int left = 0, right = 0;
    // 窗口数据结构
    int windowCount[26] = {0}; // 记录每个字符出现的次数
    int windowUniqueCount = 0; // 记录窗口中存在几种不同的字符
    int windowValidCount = 0; // 记录有多少个元素达标
    // 这时候的滑动窗口就很自然了
    while (right < s.size()) {
        char c = s[right];
        right++;

        // 更新窗口数据
        if (windowCount[c - 'a'] == 0) { windowUniqueCount++; }
        windowCount[c - 'a']++;
        if (windowCount[c - 'a'] == k) { windowValidCount++; }
        // 当字符种类大于 count 时进行收缩
        while (windowUniqueCount > count) {
            char d = s[left];
            left++;

            // 更新窗口数据
            if (windowCount[d - 'a'] == k) { windowValidCount--; }
            windowCount[d - 'a']--;
            if (windowCount[d - 'a'] == 0) { windowUniqueCount--; }
        }
        // 当字符种类等于 count 时尝试更新答案
        // 这里等价于 windowValidCount == count && windowUniqueCount == count
        if (windowValidCount == count) {
            res = max(res, right - left);
        }
    }
    return res;
}
```
