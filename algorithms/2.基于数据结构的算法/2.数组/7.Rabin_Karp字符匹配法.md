# 滑动窗口延伸：Rabin Karp 字符匹配算法
简而言之就是滑动窗口 + 动态hash匹配。

## 滑动 hash 算法
用一个例题来解释这个方法：[187.高效寻找重复子序列](https://leetcode.cn/problems/repeated-dna-sequences/)

由于限制了子串的长度，应该比较容易想到用一个大小为子串长度的“窗口”来扫一遍数组，然后记录所有出现过的子串（HashSet）。但截取子串的过程，用C++描述是`s.substr(left, L)`。这是一个$O(L)$的操作，如果扫一次子串，总时间复杂度为$O(NL)$。这个解法写成代码语言是：
```C++
// 暴力解法
vector<string> findRepeatedDnaSequences(string s) {
    int n = s.length();
    // 记录出现过的子串
    unordered_set<string> seen;
    // 记录那些重复出现多次的子串
    // 注意要用哈希集合，防止记录重复的结果
    unordered_set<string> res;

    for (int i = 0; i + 10 <= n; i++) {
        string subStr = s.substr(i, 10);
        if (seen.find(subStr) != seen.end()){
            // 之前出现过，找到一个重复的
            res.insert(subStr);
        } else {
            // 之前没出现过，加入集合
            seen.insert(subStr);
        }
    }
    return vector<string>(res.begin(), res.end());
}
```

我们能否优化这个取子串`substr`的过程呢？由于DNA只有`AGCT`这4中，因此我们可以用`0123`来分别代替。

假设我们有一个10进制数，如果要在最低添加一个数，应该如何操作？
```C++
int number = 8264;
// number 的进制
int R = 10;
// 添加在最低位的数字
int appendVal = 3;
// 在最低位添加一位
number = R * number + appendVal;
```
反之，如果我们想要将最高位的数字删除，应该如何操作？
```C++
int number = 8264;
// number 的进制
int R = 10;
// 要删除的最高位
int removeVal = 8;
// 此时，number 的长度（位数）
int L = 4
// 运算，删除最高位数字
number = number - removeVal * std::pow(R, L - 1); // 10^3
```

有这个想法，我们可以将字符串看成一串L进制数，然后通过上述操作“入队、出队”，这叫**滑动哈希技巧**。

对于上述题目，我们可以将字符串本身先改成 4 进制数，然后通过滑动 hash 技巧来解决：
```C++
vector<string> findRepeatedDnaSequences(string s) {
    // 先将字符串转化成 4 进制的数字数组
    vector<int> nums(s.size());
    for (int i = 0; i < s.size(); i++) {
        switch(s[i]) {
            case 'A':
                nums[i] = 0;
                break;
            case 'G':
                nums[i] = 1;
                break;
            case 'C':
                nums[i] = 2;
                break;
            case 'T':
                nums[i] = 3;
                break;
        }
    }
    // 记录出现过的 hash 值（4进制值）
    unordered_set<int> seen;
    // 记录重复出现的字符串结果
    // 注意，这里记录的的字符串，因此还是需要
    unordered_set<string> res;

    // 数字位数
    const int L = 10;
    // 进制
    const int R = 4;
    // 存储 R^(L-1) 的结果（最高位）
    int RL = std::pow(R, L - 1);
    // 维护滑动窗口中字符串的 hash value
    int windowHash = 0;
    
    // 滑动窗口
    int left = 0, right = 0;
    while (right < nums.size()) {
        // 扩大窗口，移入字符（最低位）
        // 维护窗口 hash 值
        windowHash = windowHash * R + nums[right];
        right++;

        // 如果满足要求，检查是否重复出现，并存入 seen
        if (right - left == L) {
            if (seen.count(windowHash)) {
                res.insert(s.substr(left, L));
            } else {
                seen.insert(windowHash);
            }
        }
        // 如果窗口过大，直接缩小窗口
        // 这个判断可以和上面写在一起，当字符串长度 == L，直接移出最高位
        // 不过为了更符合模板，这里另写一个判断
        while (right - left == L) {
            // 缩小窗口，移入字符（最低位）
            // 维护窗口 hash 值
            windowHash = windowHash - RL * nums[left];
            left++;
        }
    }
    // 转换成题目要求的 List 类型
    // 通过迭代器来写
    return vector<string>(res.begin(), res.end());
}
```

## Rabin-Karp 算法
Rabin-Karp 算法转门用来解决**字符串匹配**问题。将字符串改成容易hash的整数进行求解，避免$O(L)$的`s.substr(begin, len)`的操作。

也就是维护一个滑动窗口，记录这个滑动窗口的hash值，然后进行匹配。对于 ASCII 来说，其实就是`0~255`这些数字，可以理解成一个 256 进制的数字。用上述的移动hash来进行解决。

但由于256进制很容易整型溢出，我们可以通过取余，让这个数对`Q`取模，让这个数落在`[0, Q-1]`范围之内。让`windowHash`和`patHash`保持在这个范围内。

同时需要处理hash冲突，实际上就是判断一下两个子串是否相同即可，这时候用一个`.substr(begin, len)`截取一下即可，因为 hash 冲突不常出现，所以这里并不影响时间复杂度。
```C++
// Rabin-Karp 指纹字符串查找算法
int rabinKarp(string txt, string pat) {
    // 位数
    int L = pat.length();
    // 进制
    int R = 256;
    // 用一个比较大的素数当成取模的余数
    long Q = 1658598167;
    // R^(L-1) 的结果
    long RL = 1;
    for (int i = 1; i <= L - 1; i++) {
        // 不断求余避免溢出
        RL = (RL * R) % Q;
    }
    // 计算模式串的hash值，时间O(L)
    long pathHash = 0;
    for (int i = 1; i <= L - 1; i++) {
        pathHash = (R * patHash + pat.at(i)) % Q;
    }
    // 滑动窗口中子串字符串的hash值
    long windowHash = 0;

    // 滑动窗口代码框架，时间 O(N)
    int left = 0, right = 0;
    while (right < txt.length()) {
        // 滑动窗口，移入字符
        windowHash = (windowHash * R + txt.at(right)) % Q;
        right++;

        // 当字符串的长度达到要求
        if (right - left == L) {
            // 看是否可以匹配上
            if (windowHash == patHash) {
                // 处理 hash 冲突
                if (pat.compare(txt.substr(left, L) == 0)) {
                    return left;
                }
            }
            // 滑动窗口，移除字符
            windowHash = (windowHash - (txt.at(left) * RL) % Q + Q) % Q;
            // 这里X % Q == (X + Q) % Q 是一个模运算规则
            // 由于windowHash - (txt[left] * RL) % Q 可能出现负数，C++并不处理这个负数
            left++;
        }
    }
    return -1;
}
```

取模规则：
```C++
X % Q == (X + Q) % Q
(X + Y) % Q == (X % Q + Y % Q) % Q
```
