# 二分搜索思维
## 二分搜索问题的泛化
二分搜索能成立的基本条件，是要在题目中抽象出一个单调函数$f(x)$，以及一个目标值。然后去二分搜索这个$f(x)$。具体而言，需要能够抽象出：
1. `f(x)`必须是`x`上的单调函数
2. 题目让计算的，是满足约束条件`f(x)==target`时的$x$的值。

想象一个单调的分段函数图像，题目的要求，就是我我们求解当`f(x) == target`时$x$的下边界 / 上边界。

也就是，让我们从正面思考、直接求解（看成因变量）的 `x` 很难求解，我们这时候尝试通过二分可行域，然后通过`f(x)`单调的特点、通过**二分穷举**来求出这个`x`（或者left_bound, right_bound）。

做完上述工作在二分之前，需要有**可行域**的意识：**去找到满足题意的 `x` 的定义域，在这个定义域上去二分**。如果随便在一个取值范围上二分，很有可能让 `f(x)` 出现死循环、求解出错误的值等。

总之：当我们发现让求解的东西（有“因变量”的感觉）在**数组**上很难直接求解，但是题目中的约束条件（已经给定的条件（常量） / 另一个变量改变），求这个东西较为容易，这时候就要考虑将函数关系调转，然后看这个函数是否满足单调的特点。考虑是否二分。

注意⚠️：检查一下函数是单调递增还是单调递减的，这会影响收缩方向。

一般对于离散的数据，其实抽象出这样一个函数$f(x)$还是需要一些经验。这就需要多刷题，积累感觉了。

### [LeetCode875.爱吃香蕉的柯柯](https://leetcode.cn/problems/koko-eating-bananas/)
这个题目虽然一开始没有什么思路，但是仔细想想，有种“解空间”的感觉。对于几堆香蕉，我们要考虑在一个吃香蕉的速度`k`下，吃香蕉的时间`h`。

我们延续着解空间的感觉，如果将吃香蕉的速度当作自变量$x$，将吃香蕉的时间看成因变量$f(x)$。这时候来看题目要求：要求在一个确定的时间`h`内找可行自变量的最小值... 嘶，有点函数求解的味道！应该能想到二分搜索：

### [1011.运送货物](https://leetcode.cn/problems/capacity-to-ship-packages-within-d-days/)

与上一题感觉完全相同，还是在一个解空间中，找`f(x) = target`的left_bound。如下：

```C++
// 最大载重为 x 时能够在 f(x) 天送达所有包裹
int f(vector<int>& weights, int x) {
  int days = 0;
  for (int i = 0; i < weights.size();) {
    // 每天装载量，尽可能装更多货物
    int cap = x;
    while (i < weights.size()) {
      if (cap < weights[i]) break;
      else cap -= weigths[i];
      i++;
    }
    days++;
  }
  
  return days;
}
```

然后我们找自变量的可行域：
- 最小值`left`：数组中的最大值($\max_i{weight_i}$)；
- 最大值`right`：数组元素之和（对应一天全部装走）

然后直接进行二分，请注意：这里通过控制最小重量为`max(weights)`，来避免上述函数死循环（有完全无法装船的货物）。

注意：由于函数是单调递减的，所以这里的二分与模板中是相反的。
```C++
int shipWithinDays(vector<int>& weights, int days) {
	// 左闭右开
  int left = 0;
  int right = 1;
  // 重量最大值
  for (int w : weights) {
    left = max(left, w);
    right += w;
  }
  // 二分搜索
  while (left < right) {
    int mid = left + (right - left) / 2;
    if (f(weights, mid) < days) {
      right = mid;
    } else if (f(weights, mid) > days) {
      left = mid + 1;
    } else if (f(weights, mid) == days) {
      // 找 left_bounds
      right = mid;
    }
  }
  return left;
}
```

然后合并一下分支，最终得到：

```C++
#include <vector>
#include <algorithm>

class Solution {
public:
    int shipWithinDays(vector<int>& weights, int days) {
        int left = 0;
        int right = 1;
        for (int w : weights) {
            left = max(left, w);
            right += w;
        }

        while (left < right) {
            int mid = left + (right - left) / 2;
            if (f(weights, mid) <= days) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }

        return left;
    }

    // 定义：当运载能力为 x 时，需要 f(x) 天运完所有货物
    // f(x) 随着 x 的增加单调递减
    int f(vector<int>& weights, int x) {
        int days = 0;
        for (int i = 0; i < weights.size(); ) {
            // 尽可能多装货物
            int cap = x;
            while (i < weights.size()) {
                if (cap < weights[i]) break;
                else cap -= weights[i];
                i++;
            }
            days++;
        }
        return days;
    }
};
```



### [410.分割数组的最大值](https://leetcode.cn/problems/split-array-largest-sum/)

来看看题目：所有子数组可能性中，和最大值的最小值。

最大值的最小值，应该尝试考虑二分。但怎么分这个数组？遍历？递归？其实，如果我们将这个最大值当成一个要求解的内容来考虑，是通过某一个函数求出来的，那么：这个函数的自变量自然是子数组的最大值，那因变量是什么呢？是分割数组的个数。

如果想到这里，其实和上一个题一模一样，解法完全一样、代码完全一样！实际上我们已经解决过了。

但是为什么我们很难想出这个解法呢？是因为题目包装了一层：他让我们直接去求数组 子数组的最大值，我们最先想到的肯定是如何取划分这个数组。但我们很快发现，这个划分方法太多了，这里子数组的个数`m`虽然是定值，但我们从有多少个变量的角度看这个问题：

- 子数组的个数`m`；
- 这`m`个子数组最大值的最小值。

这个题让我们求：在一个确定的`m`下，这个最小值是多少。但我们尝试反转一下问题：让我们在确定的最大值的最小值下，去求解子数组的个数，这可以做到吗？

当然可以！在一个确定的最大值下，向后包括元素，直到大于这个最大值的最小值即可（贪心）。而且，随这个最大值的最小值的变化，子数组的个数`m`是单调递减的。到这里，我们意识到，其实这个最大值的最小值，是通过求函数的`lower_bound`来解决的...
