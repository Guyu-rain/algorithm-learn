# 差分数组

差分数组主要使用场景：频繁对原数组的**某个区间的元素进行删减**。

差分数组的定义很简单，被定义为`diff[i] = nums[i] - nums[i - 1]`，也就是本元素与前一个元素的差，如下：
```C++
int diff[nums.size()];
// 假设 "nums[-1] == 0"
diff[0] = nums[0];
for (int i = 1; i < nums.size(); i++) {
    diff[i] = nums[i] - nums[i - 1];
}
```
从差分数组 `diff[]` 是可以反推原数组 `nums[]` 的，如下：
```C++
int res[diff.size()];
res[0] = diff[0];
for (int i = 1; i < diff.size(); i++) {
    res[i] = res[i - 1] + diff[i];
}
```
> 可以发现，差分数组只涉及**本元素**和**上一个元素**，并不涉及更早的信息。这与前缀和数组是不同的。

差分数组的最大作用，是对区间进行快速增减操作。如果想让 `nums[i..j]` 中元素全部加3，我们考虑这个过程：
- 对$idx\in [i, j]$的元素，两数之差是不变化的，因此 diff[i+1..j] 不应该改动；
- 区间元素全部 `+3`，可以理解成这一个区间被“整体抬高了3”，因此，只影响和`nums[i-1]`, `nums[j+1]` 这两个元素的差值；
我们只需要：
```C++
diff[i] += 3;
diff[j + 1] -= 3;
```
原理也可以这样想：回想 `diff[]->res[]` 的过程，`diff[i] += x`意味着给 nums[i..]的所有元素都 `+x`。 
而：`diff[i] += 3` 意味着给 `nums[i..]` 所有元素都 `+3`，这时候`diff[j+1] -= 3`意味着`nums[j+1..]`所有元素都`-3`。就得到了正确的结果。


我们将差分数组抽象成一个类：
```C++
class Difference {
private:
    vector<int> diff;

public:
    // 构造差分数组
    Difference(vector<int>& nums) {
        diff = vector<int>(nums.size());
        diff[0] = nums[0];
        for (int i = 1; i < nums.size(); i++) {
            diff[i] = nums[i] - nums[i - 1];
        }
    }

    // 给区间 [i, j] 增加 val（可以为负数）
    void increment(int i, int j, int val) {
        diff[i] += val;
        // 检查边界
        if (j + 1 < diff.size()) {
            diff[j + 1] -= val;
        }
    }

    // 返回结果数组
    vector<int> result() {
        vector<int> res(diff.size());
        
        res[0] = diff[0];
        for (int i = 1; i < diff.size(); i++) {
            res[i] = res[i - 1] + diff[i];
        }
        return res;
    }
};
```

差分数组的题目一般都有一层包装，但都逃不过**每次对一个区间统一加/减**。如果能够建立这种直觉，就可以对症下药，使用差分数组来进行求解。

在使用差分数组时，需要知道数组应该开多大。也就是数据范围应该是多少。

### 差分数组的两个问题
想要用差分数组技巧，必须创建一个和区间长度一样的差分数组`diff`，如果区间非常大，对空间的开销就太大了。

而且：前缀和技巧可以快速进行区间查询，差分和数组可以快速进行区间删减，能不能将其结合起来？

其实：这两个问题的终极答案就是**线段树**，他可以在`O(logN)`的时间复杂度内完成任意长度的区间增减和区间查询操作。
