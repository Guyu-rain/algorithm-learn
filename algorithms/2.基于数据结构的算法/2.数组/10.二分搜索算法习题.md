# 二分搜索算法经典习题

## 二维数组中的二分搜索

先看一个多维坐标之间的映射转换。[566.重塑矩阵](https://leetcode.cn/problems/reshape-the-matrix/)。用`get/set`来完成这个二维数组到二维数组的映射。

其中，`get/set`都是通过在多维数组与一维数组之间的转换来完成的。

- `get`：一维索引 到 二维索引；
- `set`：同样也是一维索引到二维索引。

```C++
vector<vector<int>> matrixReshape(vector<vector<int>>& mat, int r, int c) {
    int m = mat.size(), n = mat[0].size();
    // 如果想成功 reshape，元素个数应该相同
    if (r * c != m * n) {
        return mat;
    }

    vector<vector<int>> res(r, vector<int>(c));
    for (int i = 0; i < m * n; i++) {
        set(res, i, get(mat, i));
    }
    return res;
}

int get(vector<vector<int>>& matrix, int index) {
  int m = matrix.size(), n = matrix[0].size();
  int i = index / n, j = index % n;
  return matrix[i][j];
}

void set(vector<vector<int>>& matrix, int index, int value) {
  int m = matrix.size(), n = matrix[0].size();
  int i = index / n, j = index % n;
  matrix[i][j] = value;
}
```

### [240.搜索二维矩阵II](https://leetcode.cn/problems/search-a-2d-matrix-ii/)

稍微变化了一下，其实并不是二分搜索策略。一个元素的向右、向下数字都在增加，无法简单二分。

我们的目标是尽量不走回头路。如果从左上角元素出发来考虑，那么可选的路径就是两条，肯定要涉及到回溯。但：如果从右上角（或左下角）开始搜索，就会突然发现，如果当前元素偏小，向下移动，如果当前元素偏大，向左 移动即可。这样，总共移动次数不超过`2 * N`次。如下：

```C++
bool searchMatrix(vector<vector<int>>& matrix, int target) {
    int m = matrix.size(), n = matrix[0].size();
    
    int row = 0, col = n - 1;
    // 只有两个方向：向下或者向左
    while (row < m && col >= 0) {
        int val = matrix[row][col];
        if (val == target) {
            return true;
        } else if (val < target) {
            // 让元素增大一点
            row++;
        } else if (val > target) {
            // 让元素减小一点
            col--;
        }
    }
    return false;
}
```



## 二分搜索判子序列

### [392.判断子序列](https://leetcode.cn/problems/is-subsequence/)

本题很简单，直接用双指针扫一次即可。

```C++
bool isSubsequence(string s, string t) {
    int i = 0, j = 0;
    while (i < s.size() && j < t.size()) {
        if (s[i] == s[j]) {
            i++;
        }
        j++;
    }
    return i == s.size();
}
```

### [792.匹配子序列的单词数](https://leetcode.cn/problems/number-of-matching-subsequences/)

本题中有一个字符串`s`和多个待匹配的子串`words`。主要方法还是上述的双指针，将每一个`word`尝试与`s`进行匹配。

但如果直接进行匹配，会比较慢。我们做如下处理：先将`s`的每个字符索引化。记录每个字符出现的位置`charToIndicies[c - 'a']`，然后匹配时，找指向`s`的当前指针`j`开始最近的索引。这个过程用二分进行提速。如下：

```C++
int numMatchingSubseq(string s, vector<string>& words) {
    vector<vector<int>> charToIndicies(26);
    for (int i = 0; i < s.size(); i++) {
        char c = s[i];
        charToIndicies[c - 'a'].push_back(i); 
    }

    int res = 0;
    for (string& word : words) {
        // 指向 word 的指针
        int i = 0;
        // 指向 s 的指针
        int j = 0;
        while (i < word.size() && j < s.size()) {
            // 当前 word 的字符
            char c = word[i];
            // 二分查找这个字符的位置
            if (charToIndicies[c - 'a'].empty()) break;

            int pos = left_bound(charToIndices[c - 'a'], j);
            if (pos == charToIndicies[c - 'a'].size()) break;

            j = charToIndices[c - 'a'][pos];
            // 此时已经找到
            i++;
            j++;
        }
        if (i == word.length()) {
            res++;
        }
    }
    return res;
}

int left_bound(vector<int>& arr, int target) {
    int left = 0, right = arr.size();
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (target > arr[mid]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    return left;
}
```

## 二分搜索 + 数组双指针

### [658.找到K个最接近的元素](https://leetcode.cn/problems/find-k-closest-elements/)

第一步，要找到离 `x` 最近的一个元素，将这个元素当成中心，然后通过双指针向两端移动。同时注意控制边界。

如何找到离`x`最近的一个元素？答案是通过找 `x` 的 left_bound 或 right_bound 都可以。为了简单起见，这里找 left_bound。

同时，为了可以向两端扩展，可以用双向链表来实现。或者我们用双指针维护一个区间，最后将这个区间中元素填入`vector`中也可行。

## 寻找峰值

`left_bound(nums, target)`所返回的，是`nums`中大于等于target的最小下标。如果`target`不存在，可以做如下三种等价理解：
1. 大于等于`target`的最小元素索引；
2. 是`target`应该插入在`nums`中的索引位置；
3. 是`nums`中小于`target`的元素个数。

### [162.寻找峰值](https://leetcode.cn/problems/find-peak-element/)

寻找峰值的题目，最好是考虑`nums[mid]`周围的情况。

- 如果`nums[mid] > nums[mid + 1]`**走势下行**，说明`mid`本身是一个峰值，或者其左侧有一个峰值。此时应该收缩右边界`right = mid`;
- 如果`nums[mid] <= nums[mid - 1]`**走势上行**，说明`mid`右侧有一个峰值，此时应该收缩左边界：`left = mid + 1`。

题目中说没有相等的相邻元素，因此不用考虑`nums[mid] == nums[mid + 1]`。

最终收缩停止，`[left, right)` left就是峰值。（动手分析一个例子就能理解了）

```C++
int findPeakElement(vector<int>& nums) {
    int left = 0, right = nums.size();

    while (left < right) {
        int mid = left + (right - left) / 2;

        long long midNum = static_cast<long>(nums[mid]);
        // 测试用例有一个是
        long long nextNum = mid + 1 < nums.size() ? nums[mid + 1] : -0x3f3f3f3f3f3f3f3f;

        if (midNum > nextNum) {
            right = mid;
        } else if (midNum < nextNum) {
            left = mid + 1;
        }
    }

    return left;
}
```

其中，由于取值范围是$-2^{31} <= nums[i] <= 2^{31} - 1$，也就是 `int` 的取值范围，因此表示无穷大需要用到 long。

## 特殊数组上的二分搜索

### [33.搜索旋转排序数组](https://leetcode.cn/problems/search-in-rotated-sorted-array/)

最直观的解法，是做一个映射。也就是将这个数组复原。

`idx = (idxOriginal + offset) % arrSize`，在这个数组上进行二分搜索即可。但如果这样做，需要扫一遍数组找到`offset`在哪里。

这个题目最经典的想法：将这个数组看成两端，把一个排好序的数组就好比一段斜向上的山坡，沿着一个元素旋转数组，相当于将山坡切断并旋转，在原本平滑的山坡上产生一个“断崖”。

注意「断崖」左侧的所有元素比右侧所有元素都大，我们是可以在这样一个存在断崖的山坡上用二分搜索算法搜索元素的，主要分成两步：

1. 确定 `mid` 落在断崖左侧还是右侧；
2. 在第一步确定的结果上，根据 `target` 和 `nums[left]`, `nums[right]`, `nums[mid]` 的相对大小收缩搜索区间。

首先：根据`nums[left]`和`nums[right]`之间的关系来判断`nums[mid]`落在了左 / 右。
```C++
if (nums[mid] >= nums[left]) {
    // mid 落在断崖左边，此时 nums[left..mid] 有序
} else {
    // mid 落在断崖右边，此时 nums[mid..right] 有序
}
```

假设`mid`在“断崖”左侧，那么`nums[left..mid]`是连续且有序的，如果`nums[left] <= target < nums[mid]`，则收缩右边界。否则收缩左边界。

假设`mid`在“断崖”由此，那么`nums[mid..right]`是连续且有序的，如果`nums[mid] < target <= nums[right]`，则收缩左边界，否则收缩右边界。

```C++
int search(vector<int>& nums, int target) {
    // 用左右都闭的搜索区间
    int left = 0, right = nums.size() - 1;
    // 结束条件：left > right
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 首先检查 nums[mid]，是否找到 target
        if (nums[mid] == target) {
            return mid;
        }

        // 先判断 mid 在左侧还是右侧
        if (nums[mid] >= nums[left]) {
            // mid 在左侧，nums[left..mid] 有序
            if (target >= nums[left] && target < nums[mid]) {
                // target 落在 [left..mid] 之间
                right = mid - 1;
            } else {
                // target 落在 [mid+1..right]
                left = mid + 1;
            }
        } else {
            // mid 落在右侧
            if (target <= nums[right] && target > nums[mid]) {
                // target 落在 [mid+1..right] 中
                left = mid + 1;
            } else {
                // target 落在 [left..mid-1]中
                right = mid - 1;
            }
        }
    }

    return -1;
}
```

请注意：上面表述“先判断`mid`在左侧还是右侧”其实并不标准。其实这一步是在判断到底是`nums[left..mid]`连续还是`nums[mid..right]`连续。我们优先使用`nums[left..mid]`这一段连续数组。

通过上面一步，我们拿到了一个连续递增数组，这时候，判断 `target` 是否落在这个连续数组就通过target和两端值进行比较即可。然后后面就是标准的二分。

总之：在给定的`nums[left..right]`需要先拿到一块连续递增的数组，只有在**这个区间中**才能判断`target`是否落在这个区间还是另一半区间。

### [81.搜索旋转排序数组 II](https://leetcode.cn/problems/search-in-rotated-sorted-array-ii/)

这个题多了重复元素。类似的，我们还是要尝试找到一个连续递增的区间（断崖左/右）。由于有重复元素的存在，因此我们不能简单通过上述比较(`nums[mid] 小于或大于 nums[left]`)来完成这个过程。

为了去重，画一张图`[1, 1, 1, 1, 2, 3, 0, 1, 1, 1]`来理解下面的过程。我们收缩左右边界（判断不清楚在哪一侧的重复只有可能是在两侧）进行去重（见代码）。

到这里，我们就有办法拿到一个连续递增的区间，然后执行上述第二步即可。

这个题需要画一下图，才能理解的比较清楚。

```C++
bool search(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    // 本题需要在计算 mid 之前收缩左右边界去重
    while (left < right && nums[left] == nums[left + 1]) {
        left++;
    }
    while (left < right && nums[right] == nums[right - 1]) {
        right--;
    }
    while (left <= right) {
        // 其余逻辑和第 33 题完全相同
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            return true;
        }
        if (nums[mid] >= nums[left]) {
            // mid 落在断崖左边，此时 nums[left..mid] 有序
            if (target >= nums[left] && target < nums[mid]) {
                // target 落在 [left..mid-1] 中
                right = mid - 1;
            } else {
                // target 落在 [mid+1..right] 中
                left = mid + 1;
            }
        } else {
            // mid 落在断崖右边，此时 nums[mid..right] 有序
            if (target <= nums[right] && target > nums[mid]) {
                // target 落在 [mid+1..right] 中
                left = mid + 1;
            } else {
                // target 落在 [left..mid-1] 中
                right = mid - 1;
            }
        }
    }
    // while 结束还没找到，说明 target 不存在
    return false;
}
```

### [153.寻找旋转数组中的最小值](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/)

寻找旋转数组的最小值，画一个图，发现还是去找“断崖”右侧的起点。我们有`nums[left], nums[right] 和 nums[mid]`这三个取值，我们需要将区间最后缩小到 `left == right == 断崖右侧起点`。因此，我们可以让 `nums[mid]` 和 `nums[right]` 进行比较，然后缩小区间：

- 若 `nums[mid] > nums[right]`，说明`mid`落在断崖左侧，最小值在右侧，收缩左边界：`left = mid + 1`；
- 若 `nums[mid] < nums[right]`，说明`mid`本身有可能就是最小值，且落在断崖右侧，收缩右边界(`right = mid`)。

这个题非常特殊，是判断`nums[right]`和`nums[mid]`，因此可以当成例题记住。

```C++
int findMin(vector<int>& nums) {
    int left = 0, right = nums.size() - 1;
    // 搜索区间 [left, right]
    // 直到剩下一个元素 [left, left] 时停止
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] > nums[right]) {
            // mid 落在断崖左边，最小值在 [mid+1, right]
            left = mid + 1;
        } else {
            // mid 落在断崖右边（或本身就是最小值），最小值在 [left, mid]
            right = mid;
        }
    }
    return nums[left];
}
```
