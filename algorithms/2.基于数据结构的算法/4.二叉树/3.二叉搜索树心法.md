# 二叉搜索树心法

## 特性

回忆一下二叉搜索树(BST)的定义：

1. 对于 BST 的所有节点`node`，左子树节点的值比`node`的值都要小，右子树节点的值都比 `node`值要大；
2. 对于 BST 的每一个节点`node`，它的左侧子树和右侧子树都是 BST

同时，BST 还有一个重要的性质：**BST 的中序遍历结果是有序的（升序）**

### 寻找第k小的元素

[230.二叉搜索树中第k小的元素](https://leetcode.cn/problems/kth-smallest-element-in-a-bst/)

BST 的中序遍历结果是有序的。用这个性质来解决这个问题：

```C++
// 全局变量
// 记录结果
int res = 0;
// 记录当前元素的排名
int rank = 0;

void traverse(TreeNode* root, int k) {
    if (root == nullptr) {
        return;
    } 

    traverse(root->left, k);
    // 中序位置
    rank++;
    if (rank == k) {
        res = root->val;
        return;
    }
    traverse(root->right, k);
}
```

需要注意的是，二叉搜索树如果知道以自己为根的子树一共有多少个节点（多记录一个字段），那么就能够快速知道自己在整棵树中的排序。

比如`root->size == 10`，`root->left->size == 4`，那么`root`的`rank`一定是`5`。

因此，如果要查找排名为`k`的元素，当前节点知道字节排名第`m`，那么可以比较`m`和`k`的大小：

1. 如果`m == k`，显然已经找到，直接 `return`；
2. 如果`k < m`，那么排名在第`k`的元素在左子树，这时去左子树上找第`k`个元素；
3. 如果`k > m`，那么排名第`k`的元素在右子树，这时去右子树搜索第`k - m`个元素

这样，能够将时间复杂度降低到$O(logn)$。

### BST 转化累加树

[538.把二叉搜索树转换成累加树](https://leetcode.cn/problems/convert-bst-to-greater-tree/)

最常见的想法，肯定是前缀和数组。仔细看一下题目，需要我们构造降序序列的前缀和数组。

如果中序正常遍历 BST，得到的是升序结果。如果我们能够“倒序遍历” BST，这样能够得到一个倒序遍历的结果。这时候有一个`sum`一边加一边记录就能得到正确结果。

所谓的倒序遍历，也就是先去右子树，然后去左子树即可。如下：

```C++
int sum = 0;

void traverse(TreeNode* root) {
    if (root == nullptr) {
        return;
    }
    // 倒序遍历
    traverse(root->right);
    sum += root->val;
    root->val = sum;
    traverse(root->left);
}

TreeNode* convertBST(TreeNode* root) {
    traverse(root);
    return root;
}
```

## 基础操作

在这一部分，我们实现二叉搜索树的基础操作：判断 BST 的合法性、增、删、查。

有关 BST 的问题，经常会遇到如下的代码逻辑：

```C++
void BST(TreeNode* root, int target) {
    if (root->val == target) 
        // 找到目标，做点什么
    if (root->val < target) 
        BST(root->right, target);
    if (root->val > target) 
        BST(root->left, target);
}
```

### 判断 BST 的合法性

[98.验证二叉搜索树](https://leetcode.cn/problems/validate-binary-search-tree/)

注意：二叉搜索树的定义是“所有节点的左子树节点都比本身小，所有节点的右子树都比本身大”。我们经常使用一个推论：二叉搜索树的所有子树都是二叉搜索树，这个定义本身是没有问题的，但：

如果我们有一个递归函数，判断左子树是BST && 右子树是BST，然后返回这个结果，是错误的！因为没有判断自身为根节点的树是否是BST。从而导致这个递归并不能判断BST。

因此，我们需要判断 `root` 为根的子树的所有节点是否都满足 `max->val > root->val > min->val`，递归时，需要携带`min, max`这两个节点。可以理解成**将根节点信息向下传递**：

```C++
bool isValidBST(TreeNode* root) {
    return _isValid(root, nullptr, nullptr);
}

// 该函数返回 root 为根的子树的节点是否都满足 max->val > root->val > min->val
bool _isValidBST(TreeNode* root, TreeNode* min, TreeNode* max) {
    if (root == nullptr) {
        return true; // 空节点当然是BST
    }

    // 如果 root->val 不符合 max 和 min 的限制，说明不是合法 BST
    // 需要判断根节点的合法性
    if (min != nullptr && root->val <= min->val) return false;
    if (max != nullptr && root->val >= max->val) return false;
    // 限定左子树的最大值是 root->val
    // 限定右子树的最小值是 root->val
    return _isValidBST(root->left, min, root)
        && _isValidBST(root->right, root, max);
}
```

在这种写法中，有两个值得学习的地方：

1. 通过参数，将根节点的值向下传递。下层可以获得上层的信息；对于判断 BST，向左走，约束 max 不能超过 `root->val`，向右走，约束 min 不能小于 `root->val`；
2. 需要特别判断根节点。一般二叉树的分解思路，拆成 根 + 左子树 + 右子树。其中，根才是递归中“实际做事”的地方。

### 在BST中搜索元素

[700.二叉搜索树中的搜索](https://leetcode.cn/problems/search-in-a-binary-search-tree/)

不使用 BST 特性的搜索如下，与正常二叉树的搜索相同

```C++
TreeNode* searchBST(TreeNode* root, int target) {
    if (root == nullptr) return nullptr;
    if (root->val == target) return root;
    // 当前节点没找到，递归去左右子树寻找
    TreeNode* left = searchBST(root->left, target);
    TreeNode* right = searchBST(root->right, target);
    return left != nullptr ? left : right;
}
```

考虑 BST 左小右大的特性，可以稍微改动：

```C++
TreeNode* searchBST(TreeNode* root, int target) {
    if (root == nullptr) {
        return nullptr;
    }

    // 去左子树搜索
    if (root->val > target) {
        return searchBST(root->left, target);
    }
    // 去右子树搜索
    if (root->val < target) {
        return searchBST(root->right, target);
    }
    return root;
}
```

### 在BST中插入一个数

[701.二叉搜索树中的插入操作](https://leetcode.cn/problems/insert-into-a-binary-search-tree/)

```C++
// 在以 root 为根的 BST 中插入 val 节点，返回插入后的根节点
// 题目中，保证了所有 node.val 都不相同，因此一定会去左子树或者右子树上找到一个 nullptr 插入这个节点
// 最后返回 root，就是返回这颗子树
TreeNode* insertIntoBST(TreeNode* root, int val) {
    if (root == nullptr) {
        return new TreeNode(val);
    }

    // 去右子树找插入位置
    if (root->val < val) {
        root->right = insertIntoBST(root->right, val);
    }
    // 去左子树找插入位置
    if (root->val > val) {
        root->left = insertIntoBST(root->left, val);
    }

    return root; 
}
```

### 在 BST 中删除一个数

[450.删除二叉树搜索树中的节点](https://leetcode.cn/problems/delete-node-in-a-bst/)

先找后改。找的逻辑和上述相同，分成大/小两种情况，分别去左右子树上找。

首先明确一点：我们说“删除一个数”实际上是从 BST 上将这个节点摘下来。并没有将对应的内存释放掉。

如果找到了相应的元素，就需要删除，分成三种情况：

- 如果左右子节点为空，直接删除
- 如果左右有一个节点为空，直接用子节点代替当前的位置
- 如果两边节点都不空，需要用左子树的最大节点或右子树的最小节点来替换当前节点
    - 以右子树的最小节点为例，需要从右子树根节点开始，一路向左，直到`root->left == nullptr`；然后交换要删除的节点和这个叶子节点，然后问题就转化成了删除这个最多只有一个右孩子的节点的问题了；

同时，注意到节点值是唯一的，因此查找的两个 `if` 判断必然进一个。

如下：

```C++
// 找到 BST 最小值
TreeNode* getMin(TreeNode* node) {
    while (root->left != nullptr) {
        node = node->left;
    }
    return node;
}

TreeNode* deleteNode(TreeNode* root, int key) {
    // base case
    if (root == nullptr) {
        return nullptr;
    }

    if (root->val == key) {
        // 找到，进行删除
        // 情况1 和 情况2
        // 注意：这里直接返回这个子树的处理结果
        if (root->left == nullptr) return root->right;
        if (root->right == nullptr) return root->left;
        // 情况3：先找到右子树的最小值，然后交换，最后删除
        TreeNode* minNode = getMin(root->right);
        // 删除右子树的最小节点
        root->right = deleteNode(root->right, minNode->val);
        // 用右子树最小的节点替换 root 节点
        minNode->left = root->left;
        minNode->right = root->right;
        root = minNode;
        // 可选，返回 root
    }
    // 去左子树上找这个元素
    if (root->val > key) {
        root->left = deleteNode(root->left, key);
    }
    // 去右子树上找这个元素
    if (root->val < key) {
        root->right = deleteNode(root->right, key);
    }
    // 将这颗子树正常返回上一层
    return root;
}
```

## 构造

给定一串数，我们来构造有多少个符合要求的 BST

### [96.不同的二叉搜索树](https://leetcode.cn/problems/unique-binary-search-trees/)

穷举题。如果给定 `1~n` 中一个节点作为根节点，这时可以知道左子树和右子树的节点都能取那些值。而左右子树依然是 BST，我们知道了左右子树的所有可能构造数，将其相乘就是这个根所能构造的 BST 数目。

如下，递归来写即可，需要注意的是，这本质上是一个动态规划问题，重复子问题肯定很多，因此用一个 `memo` 来进行优化。

```C++
vector<vector<int>> memo;
// 区间 [lo, hi] 作为节点能够构造的 BST 数
int count(int lo, int hi) {
    if (lo > hi) {
        return 1;
    }
    if (memo[lo][hi] != 0) {
        return memo[lo][hi];
    }

    int res = 0;
    for (int i = lo; i <= hi; i++) {
        res += count (lo, i - 1) * count(i + 1, hi);
    }
    memo[lo][hi] = res;
    return res;
}

int numTrees(int n) {
    memo = vector<vector<int>>(n + 1, vector<int>(n + 1, 0));
    return count(1, n);
}
```

### [95.不同的二叉搜索树 II](https://leetcode.cn/problems/unique-binary-search-trees-ii/)

也就是需要构造出来。类似的，还是通过递归来构造，先构造根节点，然后递归去构造左右节点。

```C++
// 构造闭区间 [lo, hi] 组成的 BST
vector<TreeNode*> build(int lo, int hi) {
    vector<TreeNode*> res;
    // base case
    if (lo > hi) {
        // 这里需要装一个 null 元素，这样才能让下面的两个内层 for 循环都能进入，正确地创建出叶子节点
        // 举例来说吧，什么时候会进到这个 if 语句？当你创建叶子节点的时候，对吧。
        // 那么如果你这里不加 null，直接返回空列表，那么下面的内层两个 for 循环都无法进入
        // 你的那个叶子节点就没有创建出来，看到了吗？所以这里要加一个 null，确保下面能把叶子节点做出来
        res.emplace_back(nullptr);
        return res;
    }

    // 1、穷举 root 节点的所有可能。
    for (int i = lo; i <= hi; i++) {
        // 2、递归构造出左右子树的所有有效 BST。
        vector<TreeNode*> leftTree = build(lo, i - 1);
        vector<TreeNode*> rightTree = build(i + 1, hi);
        // 3、给 root 节点穷举所有左右子树的组合。
        for (auto left : leftTree) {
            for (auto right : rightTree) {
                // i 作为根节点 root 的值
                TreeNode* root = new TreeNode(i);
                root->left = left;
                root->right = right;
                res.emplace_back(root);
            }
        }
    }
    
    return res;
}

vector<TreeNode*> generateTrees(int n) {
    if (n == 0) return vector<TreeNode*>{};
    return build(1, n);
}
```

## 后序遍历 / 后序位置

还是老生常谈的后序位置的好处：

>前序位置的代码只能从函数参数中获取父节点传递来的数据，而后序位置的代码不仅可以获取参数数据，还可以获取到子树通过函数返回值传递回来的数据。
> 
> 那么换句话说，一旦你发现题目和子树有关，那大概率要给函数设置合理的定义和返回值，在后序位置写代码了。

经常，我们在后序位置处理节点，同时更新答案，能够大幅提升算法效率。

**如果当前节点要做的事情需要通过左右子树的计算结果推导出来，就要用到后序遍历。**

### [二叉搜索子树的最大键值和](https://leetcode.cn/problems/maximum-sum-bst-in-binary-tree/)

站在每个节点上，需要知道：

1. 左右子树是否是 BST
2. 左子树的最大值和右子树的最小值
3. 左右子树的节点值之和

为了知道如下的信息，最朴素的想法，是每一个值节点都去遍历一次子树，有如下伪代码：

```C++
int maxSum = 0;

// 遍历二叉树
void traverse(TreeNode* root) {
    if (root == NULL) {
        return;
    }

    // ******* 前序位置 *******
    // 判断左右子树是不是 BST
    if (isBST(root->left) && isBST(root->right)) {
        // 计算左子树的最大值和右子树的最小值
        int leftMax = findMax(root->left);
        int rightMin = findMin(root->right);
        // 判断以 root 节点为根的树是不是 BST
        if (root->val > leftMax && root->val < rightMin) {
            // 如果条件都符合，计算当前 BST 的节点之和
            int leftSum = findSum(root->left);
            int rightSum = findSum(root->right);
            int rootSum = leftSum + rightSum + root->val;
            // 计算 BST 节点的最大和
            maxSum = max(maxSum, rootSum);
        }
    }
    // **************************

    // 二叉树遍历框架，遍历子树节点
    traverse(root->left);
    traverse(root->right);
}

// 计算以 root 为根的二叉树的最大值
int findMax(TreeNode* root) {}

// 计算以 root 为根的二叉树的最小值
int findMin(TreeNode* root) {}

// 计算以 root 为根的二叉树的节点和
int findSum(TreeNode* root) {}

// 判断以 root 为根的二叉树是否是 BST
bool isBST(TreeNode* root) {}

int maxSumBST(TreeNode* root) {
    traverse(root);
    return maxSum;
}
```

这几个辅助函数都是递归函数，都要遍历输入的二叉树，外加 `traverse` 本身的递归，复杂度是非常高的。

我们将 `traverse` 这个函数改成后序遍历，让其将辅助函数做的事情都顺便做掉。如果我知道了左右子树的最大值，就可以推导出以我为根的这棵树的最大值，并不需要遍历所有节点。同理，最小节点也是，只需要知道左右子树的最小节点即可。

我们让 `findMaxMinSum` 这个函数做一些计算任务，返回一个大小为 4 的 int 数组`res`，其中：

- `res[0]`记录以`root`为根的二叉树是否为 BST
- `res[1]`记录以`root`为根的二叉树所有节点的最小值
- `res[2]`记录以`root`为根的二叉树所有节点的最大值
- `res[3]`记录以`root`为根的二叉树所有节点值之和

有了这个优化，对于每个节点计算出左右子树的这 4 个值，只需要简单运算，就能推导当前节点的情况，避免了重复遍历。

```C++
int maxSum = 0;

vector<int> findMaxMinSum(TreeNode* root) {
    // base case
    // 空节点，此时是 BST，后面在比较最小、最大值时需要放行，因此用最大最小值
    if (root == nullptr) {
        // 不需要更新 maxSum
        return {1, INT_MAX, INT_MIN, 0};
    }

    // 我们去后序位置写代码，这里拿到左右子树的情况
    vector<int> leftRes = findMaxMinSum(root->left);
    vector<int> rightRes = findMaxMinSum(root->right);

    // 在后序位置，对这几个值进行计算
    vector<int> res(4);

    // 判断是否是 BST，同时更新 maxSum
    // 如果左右子树都是 BST，同时根节点值 > 左子树最大值 且 < 右子树最小值
    if (leftRes[0] && rightRes[0] 
        && root->val > leftRes[2]
        && root->val < rightRes[1]) {
        res[0] = 1;
        // 最小值，左子树的最小值 或 根节点值
        res[1] = min(leftRes[1], root->val);
        // 最大值，右子树的对大值 或 跟节点值
        res[2] = max(rightRes[2], root->val);
        res[3] = leftRes[3] + rightRes[3] + root->val;
        maxSum = max(res[3], maxSum);
    } else {
        // 以 root 为根的二叉树不是 BST
        // 只需要知道 res[0] = 0，其余的不用计算
        res[0] = 0;
    }
    return res;
}

int maxSumBST(TreeNode* root) {
    findMaxMinSum(root);
    return maxSum;
}
```

**如果一个问题是分解思路，那么一般要配合后序遍历的方式。因为我们需要的所有变量都可以在后序的位置上找到。**

实际上，这句话也应该反过来看：如果一个题目对某一节点的计算依赖左右子树所计算获得的结果，在后序位置进行处理然后返回，那我们就需要用分解问题的思路；

**同时：尽可能避免递归函数中调用递归。**这是一个很重要的直觉。如果发现主要求解逻辑中需要调用一些状态（比如本题中左右子树的最小、最大、和等），这些状态需要递归来解，那么可以用本题的技巧，**用一个结构化数据存储这些状态**，同时在**后序**的位置处理这些值，得到正确的结果。
