# 反转链表
在LeetCode中，单链表定义如下：
```C++
class ListNode {
public:
    int val;
    ListNode *next;
    ListNode(int x) : val(x), next(nullptr) {}    
};
```
到这里，栈似乎是一个不错的数据结构，但需要太多的额外空间～

## 反转整个链表
### 迭代解法
```C++
ListNode* reverseList(ListNode* head) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }
    // 由于单链表的结构，至少需要三个指针才能完成迭代反转
    ListNode *pre, *cur, *nxt;
    pre = nullptr; cur = head; nxt = head->next;
    while (cur != nullptr) {
        // 逐个节点反转
        cur->next = pre;
        // 更新指针位置
        pre = cur;
        cur = nxt;
        if (nxt != nullptr) {
            nxt = nxt->next;
        }
    }
    return pre;
}
```
这个解法用三个指针，分别维护 `cur`, `pre`, `nxt`，每个位置如下：
```
   node(i) -> node(i+1) -> node(i+2)
     pre         cur          nxt
```
需要将其转换为：
```
   node(i) <- node(i+1)    node(i+2)
     pre         cur          nxt
```
然后指针向后不停移动。这个方法要特别小心边界的处理：
- 一旦出现`nxt.next`这种操作，就要条件反射想到：是否会出现 `nxt` 本身是 `nullptr`；
- 注意循环中止的条件，如果不能清楚何时停止，可以用一个最简单的例子：`1->2`来理解。

### 递归解法
反转链表这个问题是有子问题的，如果我们要反转单链表`1->2->3->4`，假设我们已经有了`reverse(2->3->4)`的结果，那么我们只需要想办法把`1`放在这个反转后链表的最后：
```C++
reverseList(1->2->3->4) = reverse(2->3->4)->1
```
然后返回这个链表即可。

**这是“分解问题”的思路，通过递归函数的定义，将原问题分解成若干规模更小、结构相同的子问题，最后通过子问题的答案组装出原问题的解。**

通过这个想法，有解如下：
```C++
ListNode* reverseList(ListNode* head) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }
    // 找一个子问题的解
    ListNode* last = reverseList(head->next); // 1
    // 拿到 last 之后，将 head 拼在后面
    // 这一步还是很有技巧性的
    head->next->next = head; // 2
    head->next = nullptr; // 3
    return last;
}
```
我们来看这几步都发生了什么：
1. 我们获得了反转的链表，从：
```C++
 1->(2->3->4->nullptr)
```
变成了：
```C++
1->(2<-3<-4)
    ↓
  nullptr
```
注意这里：1和后面的子问题`reverse(2->...)`传入的头节点`2`还是藕断丝连，1.next 结果是2。
2,3. 我们要将`1`拼到`2`的后面。已知：`1.next`是`2`，怎么将`1`拼接到`2`后面？
- 先拿到`2`这个节点：`1.next`；
- 然后将其`next`变成`1`：`1.next.next = 1`；
- 然后断开 `1->2` 这一个连接。
也就是上面2，3两步。

最终，返回反转后的头节点`last`。

## 反转链表前 N 个节点
### 递归解法
还是维护三个节点，只不过需要固定步数后停止：
> 要能够想清楚 N 步之后链表长成什么样！
```C++
ListNode* reverseN(ListNode* head, int n) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }

    ListNode *pre, *cur, *nxt;
    pre = nullptr, cur = head, nxt = head->next;
    
    // 反转前 N 个节点
    for (int i = 0; i < n; i++) {
        // 反转链表
        cur->next = pre;
        // 向后移动
        pre = cur;
        cur = nxt;
        if (nxt != nullptr) {
            nxt = nxt->next;
        }
    }
    // 此时获得的链表长这样：
    // nullptr<-1(head)<-2<-...<-N-1<-N(pre)   N+1(cur)->...->end->nullptr
    // 我们需要将从 N+1 开始的链表拼接到 1 后面
    head->next = cur;    
    return pre;
}
```

### 递归解法
类似的，我们还是要找一个子问题：
```C++
reverseList(1->2->3->4) ->5->6 = reverse(2->3->4)->1 ->5->6
```
在解题过程中，我们发现子问题解决需要记录“不需要更改部分的起点”，否则链表会直接断开；
```C++
ListNode* successor = nullptr; // 记录不需要更改部分的头节点

ListNode* reverseN(ListNode* head, int n) {
    if (n == 0 || n == 1) {
        // 记录 n + 1 个节点
        if (n == 1) {
            successor = head->next;
        }
        return head;
    }

    ListNode* last = reverseN(head->next, n - 1);
    // 此时的链表长成这样：
    // 1->2<-3<-4 5->6
    //    ↓
    //  nullptr
    // 链表直接被断开，我们需要记录 5
    head->next->next = head;
    head->next = successor;

    return last;
}
```
注意区别：
1. base case 变成`n == 1`（反转一个元素），就是本身，直接返回。此时需要记录后面的节点，就是`n + 1`个节点（不需要改变的第一个节点）；
2. 子问题解决完后，注意`head->next`应该指向不需要改变的后半段链表；

## 反转链表的一部分[LeetCode92.反转链表 II](https://leetcode.cn/problems/reverse-linked-list-ii/)
### 迭代解法
先找到`right`节点，然后反转前面`right - left + 1`个节点，返回即可。
```C++
ListNode* reverseBetween(ListNode* head, int m, int n) {
    if (m == 1) {
        return reverseN(head, n);
    }
    // 找到 m 个节点的前驱
    ListNode* pre = head;
    for (int i = 1; i < m - 1; i++) {
        pre = pre->next;
    }
    // 拿到了前驱，反转后面链表的前 n - m + 1个元素，然后直接拼回来
    pre->next = reverseN(pre->next, n - m + 1);
    return head;
}

ListNode* reverseN(ListNode* head, int n) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }

    ListNode *pre, *cur, *nxt;
    pre = nullptr, cur = head, nxt = head->next;
    
    // 反转前 N 个节点
    for (int i = 0; i < n; i++) {
        // 反转链表
        cur->next = pre;
        // 向后移动
        pre = cur;
        cur = nxt;
        if (nxt != nullptr) {
            nxt = nxt->next;
        }
    }
    // 此时获得的链表长这样：
    // nullptr<-1(head)<-2<-...<-N-1<-N(pre)   N+1(cur)->...->end->nullptr
    // 我们需要将从 N+1 开始的链表拼接到 1 后面
    head->next = cur;    
    return pre;
}
```

## [LeetCode25.K 个一组反转列表🔥](https://leetcode.cn/problems/reverse-nodes-in-k-group/)
到现在，我们手头有很高级的工具：反转前 N 个元素(function: `reverseN(ListNode*, int)`)：
```C++
ListNode* reverseN(ListNode* head, int n) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }

    ListNode *pre, *cur, *nxt;
    pre = nullptr, cur = head, nxt = head->next;
    
    // 反转前 N 个节点
    for (int i = 0; i < n; i++) {
        // 反转链表
        cur->next = pre;
        // 向后移动
        pre = cur;
        cur = nxt;
        if (nxt != nullptr) {
            nxt = nxt->next;
        }
    }
    // 此时获得的链表长这样：
    // nullptr<-1(head)<-2<-...<-N-1<-N(pre)   N+1(cur)->...->end->nullptr
    // 我们需要将从 N+1 开始的链表拼接到 1 后面
    head->next = cur;    
    return pre;
}
```
最朴素的想法，是正向思考：先反转前 K 个，然后再向后 K 个，直到不足 K 个。
为了处理“不足 K 个时直接返回”，还是可以用快慢指针思想，只要快指针 nullptr 了，就直接返回：
```C++
ListNode* reverseKGroup(ListNode* head, int k) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }
    ListNode* prev = new ListNode(-1, head);
    ListNode *fast = head;
    // 第一次反转，需要记录头节点，因此单独来写
    for (int i = 0; i < k; i++) {
        if (fast == nullptr) {
            return head;
        }
        fast = fast->next;
    }
    prev->next = reverseN(prev->next, k);
    head = prev->next; // 将头节点记录下来，准备返回

    // 要一个前驱
    
    while (true) {
        for (int i = 0; i < k; i++) {
            if (fast == nullptr) {
                return head;
            }
            prev = prev->next;
            fast = fast->next;
        }
        prev->next = reverseN(prev->next, k);;
    }

    return head; // 可有可无
}
```
时刻注意⚠️：`reverseN`的行为，是将`head`向后的`n`个节点反转，并返回新的节点。#**如果想要链表能够“拼起来”，必须要记录其前驱节点！**否则会单链表无法回头溯源上一个节点是什么！

### 这个问题其实有递归性质
比如我们对如下这个链表调用`reverseKGroup(head, 2)`：
```C++
(head)
   1->2->3->4->5->6->nullptr
```
问题解决应该是先反转前两个节点，后面仍然是一条链表，然后反复调用`reverseKGroup(newHead, 2)`。总结一下解题思路，应该是：
1. 先反转`head`开头的`k`个元素，使用`reverseN`函数；
2. 将 `k+1` 个元素作为`head`递归调用`reverseKGroup`函数；
3. 将1.反转的前两个元素与2.得到的链表拼接起来，最终得到结果；
```C++
ListNode* reverseKGroup(ListNode* head, int k) {
    if (head == nullptr) {
        return nullptr;
    }
    // 区间 [a, b) 包含 k 个带反转元素
    ListNode *a, *b;
    a = b = head;
    for (int i = 0;  i < k; i++) {
        // 不足 k 个，直接 return
        if (b == nullptr) return head;
        b = b->next;
    }
    //反转前 k 个元素
    ListNode *newHead = reverseN(a, k);
    // 此时 b 指向下一组带反转的头节点
    // a 反转后称为链表尾部
    // 递归反转后序链表，然后拼接起来
    a->next = reverseKGroup(b, k);
    return newHead;
}

// reverseK
ListNode* reverseN(ListNode* head, int n) {
    if (head == nullptr || head->next == nullptr) {
        return head;
    }

    ListNode *pre, *cur, *nxt;
    pre = nullptr, cur = head, nxt = head->next;

    // 向后反转 n 个
    while (n--) {
        cur->next = pre;
        // 指针后移
        pre = cur;
        cur = nxt;
        if (nxt != nullptr) {
            nxt = nxt->next;
        }
    }

    // 此时链表长成：
    // nullptr<-1(head)<-...<-N(pre)  N+1(cur)->N+2(nxt)->...->end->nullptr
    head->next = cur;
    return pre;
}
```
