# `LRU 算法` 与 `双向哈希链表`

## [LeetCode146.LRU缓存机制](https://leetcode.cn/problems/lru-cache/)

LRU 指的是 Least Recently Used，我们认为最近使用过的数据应该是最“有用的”。我们优先保留他们，优先淘汰“最长时间没有出现过的”。

我们要实现`get`，`put`这两个方法，而这两个方法都需要是 $O(1)$ 的时间复杂度。

### 设计 LRU 算法的必要条件：

1. `cache` 中的元素必须有时序，以区分最近使用的和久未使用的数据。当容量满后要删除最久未使用的元素；
2. 我们要在`cache`中快速找到`key`是否已经存在，并得到对应的`val`；
3. 每次访问`cache`中的某个`key`，需要将这个元素变为最近使用的，也就是说`cache`要支持在任意位置插入和删除元素；

### 哈希链表：`LinkedHashMap`

LRU 缓存的核心数据结构就是哈希链表，是双向链表和哈希表的结合体。

- 由一个`双向链表`和一个`hashmap`组成;
- `双向链表` 被分成了两个字段：`key`, `val`；
- `hashmap`记录了`key`的位置。

每次默认从链表尾部添加元素，也就是越靠近链表尾部的元素越是最近使用过的；每一次访问都需要能够快速定位到对应的`key`。

### 代码实现

我们首先实现**双向链表**；

**双向链表节点**定义如下：

```C++
class Node {
public:
    int key, val; // 两个字段
    Node* next;
    Node* prev;
    Node(int k, int v) : 
        key(k), val(v) {}
    Node(int k, int v, Node* nxt, Node* pre) : 
        key(k), val(v), next(nxt), prev(pre) {}
};
```

定义双向链表，同时定义几个操作：
```C++
class DoubleList {
private:
    // 头尾虚拟节点
    Node* head;
    Node* tail;
    // 链表元素个数
    int size();

// 暴露几个接口
public:
    DoubleList() {
        head = new Node(0, 0);
        tail = new Node(0, 0);
        head->next = tail;
        tail->prev = head;
        size = 0;
    }

    // 在链表尾部添加节点，时间复杂度 O(1)
    void addLast (Node* x) {
        x->next = tail;
        x->prev = tail->prev;
        tail->prev->next = x;
        tail->prev = x;
        size++;
    }

    // 删除链表中的 x 节点（x 一定存在）
    // 由于是双链表，且给定目标是 Node 节点，时间 O(1)
    void remove(Node* x) {
        x->prev->next = next;
        x->next->prev = prev;
        size--; 
    }

    // 删除链表中的第一个节点，并返回该节点，时间 O(1)
    Node* removeFirst() {
        if (head->next == tail) {
            return nullptr;
        }
        Node* first = head->next;
        remove(first);
        return first;
    }

    // 返回链表长度，时间 O(1)
    int getSize() { return size; }
};
```

有了这个双向链表，我们需要将其与 HashMap 结合起来，得到题目要求的类；

我们需要同时维护两个数据结构，很容易误操作。为了解决这个问题，我们在这两种数据结构之上**提供一层抽象API**。

就是尽量让 LRU 的主方法 `get`, `put` 避免直接操作 `map` 和 `cache` 的细节。

#### `get` 方法需要用到的操作

1. 查询是否存在，如果不存在，返回 -1；
2. 将该数据提升为最近使用的。这个操作我们抽象成`makeRecently`。

#### `put` 方法需要的操作

`put`方法略微复杂。

1. 如果`key`已经存在，将val更改(删除`key` `deleteKey`，然后添加到最新`addRecenlty`)；
2. 如果`key`不存在，此时需要插入`key`
    1. 如果容量未满，插入为最近使用过的数据(`addRecently`)；
    2. 如果容量已满，淘汰最久未使用过的`removeLeastRecently`，然后插入(`addRecently`)

综上，我们抽象出4个对数据结构的操作：

1. `makeRecently`：将某个 `key` 提升为最近使用的，`get`方法用；
2. `addRecently`：添加最近使用的元素，`put`方法用；
3. `deleteKey`：删除某一个 `key`，`put` 方法用；
4. `removeLeastRecently`：淘汰最久未使用过的`key`，`put`方法用。

综上，如下：

```C++
// 省略 Node 和 DoubleList 
class LRUCache {
private:
    // key -> Node(key, val)
    unordered_map<int, Node*> map;
    // 双向链表，(key, val)
    DoubleList cache;
    // 最大容量
    int cap;
public:
// 定义几个 API，隐藏数据结构细节，用来操作 map 和 cache
    // 将已经存在的 Node 改成最新的节点
    void makeRecently(int key) {
        Node* x = map[key];
        // 先删除，不需要删除 map 的映射，还是指向这个节点即可
        cache.remove(x);
        // 然后将节点放在最新的位置
        cache.addLast(x);
    }
    // 将已经存在的 Node 删除
    void deleteKey(int key) {
        cache.remove(map[key]);
        map.erase(key);
    }
    // 新建一个节点
    void addRecently(int key, int val) {
        Node* x = new Node(key, val);
        map[key] = x;
        cache.addLast(x);
    }
    // 删除最不常用的节点
    void removeLeastRecently() {
        Node* deletedNode = cache.removeFirst();
        int deleteKey = deletedNode->key;
        map.erase(deletedKey);
    }

    LRUCache (int capacity) {
        this->cap = capacity;
    }

    int get(int key) {
        if (!map.contains(key)) {
            return -1;
        }
        // 如果存在，将其变成最新的，然后返回 val
        makeRecently(key);
        return map[key]->val;
    }

    void put(int key, int value) {
        // 如果 key 已经存在，
        // 删除旧的数据，
        // 然后将插入的数据作为最近使用的数据
        if (map.count(key)) {
            deleteKey(key);
            addRecently(key, val);
            return;
        }
        // 如果 key 不存在
        // 先判断是否需要删除
        if (cache.getSize() == cap) {
            // 删除最不常用的
            removeLeastRecently();
        }
        // 然后添加元素
        addRecently(key, value);
    }
};
```
