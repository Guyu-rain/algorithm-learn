# LFU 算法

LFU : Least Frequently Used，淘汰使用次数最少的数据；

LFU 算法需要模拟：删除访问元素频次最少的元素，如果两个元素访问频次相同，还要通过访问时间排序；

## [460.LFU缓存](https://leetcode.cn/problems/lfu-cache/)

### 思路分析

我们先列举算法执行过程中显而易见的几个事实：

0. 设计节点时，应该有一个字段`freq`，用来表示该节点的访问次数；
1. 调用 `get(key)` 方法时，要返回 `key` 对应的 `val`；
2. 只要用 `get` 或者 `put` 访问某个节点，`freq`字段就要 `+1`；
3. 如果再容量满之后进行插入，`freq` 最小对应的节点就应该被删除，如果有相同的`freq`，就要用时间先后break tie。

为了满足上述三个条件，我们需要：

1. 为了让 get(key) 的复杂度为 $O(1)$，使用一个 `HashMap` 存储 `key` 到 `val` 的映射：

```C++
unordered_map<int, int> keyToVal;
```

2. 为了能够对 `freq` 字段快速操作，使用一个 `HashMap` 存储 `key` 到 `freq` 的映射：

```C++
unordered_map<int, int> keyToFreq;
```

3. 这个需求是 LFU 算法的核心，

- 首先，肯定需要`freq`到`key`的映射，用来找到`freq`最小的`key`；
- 将`freq`最小的`key`删除，肯定不能遍历查找最小的`freq`；
- 可能有多个 `key` 拥有相同的 `freq`，所以`freq`与`key`是一对多的关系；也就是需要一个`freq`对应一个`key`列表；
- 同时希望 `freq` 对应 `key` 列表是存在时序的，便于快速查找并删除最旧的`key`；
- 能够快速删除`key`列表中任何一个`key`：如果某个`key`被访问，就需要将其频次变成`freq + 1`。也就是将其删除，然后放入`freq + 1`的列表中

```C++
unordered_map<int, list<int>> freqToKeys;
int minFreq = 0;
```

这实际上就是一个 `LinkedHashSet`。能够用 $O(1)$ 的时间复杂度找到 `freq` 对应的 `keys`。同时，能用 $O(1)$ 的时间复杂度访问、删除头、尾元素。同时还能够保证同一个`freq`下的访问先后顺序不变。

### 代码框架

我们将上述三张表分别称为：`KV`，`KF`, `FK`表。设计复杂数据结构时，要：

1. 自顶向下，逐步求精，先写清楚住函数的逻辑框架，然后再一步步实现细节；
2. 搞清楚映射关系，如果更新了某个`key`对应的`freq`，要同步修改 `KF` 表和 `FK` 表；
3. 画图，将复杂的逻辑关系用流程图画出来；

基本的数据结构如下：

```C++
#include <unordered_map>
#include <unordered_set>

class LFUCache {
    // key 到 val 的映射，我们后文称为 KV 表
    unordered_map<int, int> keyToVal;
    // key 到 freq 的映射，我们后文称为 KF 表
    unordered_map<int, int> keyToFreq;
    // freq 到 key 列表的映射，我们后文称为 FK 表
    unordered_map<int, list<int>> freqToKeys;
    // 记录最小的频次
    int minFreq;
    // 记录 LFU 缓存的最大容量
    int cap;

public:
    LFUCache(int capacity) {
        keyToVal.clear();
        keyToFreq.clear();
        freqToKeys.clear();
        this->cap = capacity;
        this->minFreq = 0;
    }

    int get(int key) {}

    void put(int key, int val) {}
};
```

先设计 `get` 方法：

```C++
int get(int key) {
    if (keyToVal.find(key) == keyToVal.end()) {
        return -1;
    }
    // 能找到，增加 key 对应的 freq
    increaseFreq(key);
    return keyToVal[key];
}
```

核心逻辑是增加 `key` 对应的 `freq`，将其抽象成一个函数；

设计 `put` 方法。逻辑略微复杂，画一个图来梳理逻辑。用语言描述，大概是：

先判断 `key` 是否已经存在:

1. 若 `key` 已经存在，修改 `key` 对应的 `val`，增加 `key` 对应的 `freq`。
2. 若 `key` 不存在，需要新插入 `key`；

    此时，需要判断容量是否已满

    1. 如果容量满：淘汰 `freq` 最小的 `key` 然后插入 `key`；
    2. 如果未满，直接插入。

```C++
void put(int key, int val) {
    if (this->cap <= 0) {
        return;
    }
    // 如果 key 已经存在，修改对应的 val 即可
    if (keyToVal.find(key) != keyToVal.end()) {
        keyToVal[key] = val;
        // key 对应的 freq +1
        increaseFreq(key);
        return;
    }
    // key 不存在，需要插入
    // 如果容量已经满，则需要淘汰一个 freq 最小的 key
    if (keyToVal.size() >= this->cap) {
        removeMinFreqKey();
    }
    // 插入 key 和 val，对应的 freq == 1
    // 插入 KV 表
    keyToVal[key] = val;
    // 插入 KF 表
    keyToFreq[key] = 1;
    // 插入 FK 表
    freqToKey[1].insert(key);
    this->minFreq = 1;
}
```

### LFU 核心逻辑

下面来实现抽象出的几个函数：

1. `removeMinFreqKey`：删除频率最小的元素

```C++
int removeMinFreqKey() {
    auto& keyList = freqToKeys[this->minFreq];
    // 维护 FK 表，注意 FK 表还有一个 minFreq 字段
    int deletedKey = keyList.front();
    keyList.pop_front();
    if (keyList.empty()) {
        freqToKeys.erase(this->minFreq);
        // 如果 freqToKeys[minFreq] 为空，需要更新 minFreq
    }
    // 维护 KV 表中的 key
    keyToVal.erase(deletedKey);
    // 维护 KF 表中的 key
    keyToFreq.erase(deletedKey);
}
```

有一个显而易见的问题，如果 deletedKey 这个 LinkedList 空了，就需要设置 minFreq 这个字段。但我们反过来看什么时候回调用 `removeMinFreqKey` 这个函数，发现是队满时需要调用。而新插入的`key`对应的`freq`一定为 1。`minFreq` 这个字段会在那个时候被设置。

2. `increaseFreq`：给`key`频率增加`1`

```C++
void increaseFreq(int key) {
    int freq = keyToFreq[key];
    // 更新 KF 表
    keyToFreq[key] = freq + 1;
    // 更新 FK 表
    auto& oldlist = freqToKeys[freq];
    // 将 key 从 freq 对应的列表中删除
    oldList.remove(key);
    // 将 key 加入 freq + 1 对应的列表中
    freqToKeys[freq + 1].push_back(key);
    // 维护 FK 表，如果 freq 对应的列表空，移除这个 freq
    if (old.empty()) {
        freqToKeys.erase(freq);
        // 注意更新 minFreq
        if (freq == this->minFreq) {
            this->minFreq++;
        }
    }
}
```

将以上代码组合起来，就是最终的`LFU Cache`啦～
