# 哈希表更多习题

### [1.两数之和](https://leetcode.cn/problems/two-sum/)

双指针是比较好的解法。还有一种思路就是用哈希表辅助判断，对于一个元素 `nums[i]`，看一下`target - nums[i]` 在不在这个哈希表中即可。

```C++
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> valToIndex;
    for (int i = 0; i < nums.size(); i++) {
        // 查表
        int need = target - nums[i];
        if (valToIndex.find(need) != valToIndex.end()) {
            return {valToIndex[need], i};
        }
        valToIndex[nums[i]] = i;
    }
    return {};
}
```

### [128.最长连续序列](https://leetcode.cn/problems/longest-consecutive-sequence/)

题目要求不排序，需要空间换时间。将数组元素放在 hashmap 中，然后去寻找连续序列的第一个元素，也就是拿到一个元素`num`之后，看一下 `num + 1` 在不在序列中。

我们需要首先将元素全部放入一个 hashSet 中，方便查找。这也是空间换时间的体现。

```C++
int longestConsecutive(vector<int>& nums) {
    unordered_set<int> set(nums.begin(), nums.end());

    int res = 0;
    // 找最长序列
    // 注意：要去遍历 set 而不是 nums，set 同时会去重
    for (const int& num : set) {
        // 必须是某序列的开头，否则肯定有更长的
        if (set.find(num - 1) != set.end()) {
            continue;
        }

        int curNum = num;
        int curLen = 1;
        
        while (set.find(curNum + 1) != set.end()) {
            curNum += 1;
            curLen += 1;
        }
        
        res = max(res, curLen);
    }
    return res;
}
```

注意：去遍历 set，不要遍历 nums，set还会去重，当数组中元素有大量重复时，遍历 nums 会超时。

### [290.单词规律](https://leetcode.cn/problems/word-pattern/)

其实就是判断映射关系是否成立，只不过细节比较多。下面学习一种分割字符串的方法：

```C++
bool wordPattern(string pattern, string s) {
    vector<string> words;
    size_t start = 0;
    size_t end = s.find(' ');
    // 一种字符串分割方法
    while (end != string::npos) {
        word.push_back(s.substr(start, end - start));
        start = end + 1;
        end = s.find(' ', start);
    }
    // 注意，此时最后一个单词并没有被读入
    words.push_back(s.substr(start));

    // 有些测试用例字符个数和单词个数不同
    if (pattern.length() != words.size()) {
        return false;
    }

    // 然后进行匹配
    // 记录 pattern 字符到单词的映射
    unordered_map<char, string> patternToWord;
    // 记录已经有哪些 pattern 对应的单词
    unordered_set<string> wordSet;

    for (int i = 0; i < pattern.length(); i++) {
        char c = pattern[i];
        string word = words[i];
        // 这个 c 没有被匹配过
        if (patternToWord.find(c) == patternToWord.end()) {
            // 但是这个 word 没有出现过
            if (wordSet.find(word != wordSet.end())) {
                // 说明这个单词失配了
                return false;
            }
            // 这个单词以及这个 pattern 是第一次出现，记录下来
            patternToWord[c] = word;
        } else {
            // 这个 pattern 字符已经出现过，需要确保和之前对应的单词相同
            if (patternToWord[c] != word) {
                return false;
            }
        }
        // 这个单词已经有模式与之匹配
        wordSet.insert(word);
    }
    return true;
}
```

### [519.随机翻转矩阵](https://leetcode.cn/problems/random-flip-matrix/)

这个题的本质，是将一个长度为 `m*n` 的数组，每次从中随机抽取一个位置，然后删除他。这个删除的过程其实就是与当前的最后一个元素进行调换，然后每次删除最后一个元素。

我们用 `len` 记录当前数组长度，用一个 `hashMap deletedToExist` 来记录删除元素的映射关系。什么意思呢？

> 比如当前的矩阵是 [1, 2, 3, 4, 5, 6]，`len = 6`。 我们第一次选择了元素 2，此时要将这个元素进行删除，那么我们就记录 `deletedToExist[2] = 6`，也就是记录最后 2 这个位置的元素被映射到了哪里，然后将 `len -= 1`。

于是，在随机选择一个数`num`之后，我们先查看是否被删除 (`deletedToExist.count(num)`)，如果这个已经被删除，那么我们需要反复查找还没有被删除的元素。如下：

```C++
class Solution {
private:
    int m, n;
    // 记录一维数组的长度
    int len;
    // 已删除的元素到尾部未删除元素的映射
    unordered_map<int, int> deletedToExist;

public:
    Solution(int m, int n) : m(m), n(n), len(m * n) {}

    vector<int> filp() {
        random_device rd;
        mt19937 gen(rd());
        int rand = gen() % len;
        // 选择了一个随机数，但是可能已经被删除掉了
        int res = rand;
        if (deletedToExist.count(rand)) {
            // 如果已经被删除掉，去找这个位置被映射到了哪里
            res = deletedToExist[rand];
        }
        // 将 rand 置换到数组尾部
        int last = len - 1;
        // 尾部元素可能已经被删除掉了，也需要挂一层映射
        if (deletedToExist.count(last)) {
            res = deletedToExist[last];
        }
        // 然后将现在要删除的元素映射到尾部
        deletedToExist[rand] = last;
        // 将尾部的元素删掉
        len--;
        // 一维坐标化二维
        return {res / n, res % n};
    }

    void reset() {
        // 其实就是将映射关系清空即可
        len = m * n;
        deletedToExist.clear();
    }
};
```

### [138.随机链表的复制](https://leetcode.cn/problems/copy-list-with-random-pointer/)

**对于数据结构的复制，都是：一个hashmap + 两次遍历**。

1. 第一次遍历：专门克隆节点，借助 hashMap 将原始节点和克隆节点的映射存储起来；
2. 第二次遍历：专门组装节点，按照原数据结构的形式，将克隆节点的指针组装起来。

```C++
Node* copyRandomList(Node* head) {
    unordered_map<Node*, Node*> originToClone;
    // 第一次遍历，将所有节点 clone 出来
    for (Node* p = head; p != nullptr; p = p->next) {
        if (originToClone.find(p) == originToClong.end()) {
            originToClone[p] = new Node(p->val);
        }
    }
    // 然后将节点结构构造出来
    if (Node* p = head; p != nullptr; p = p->next) {
        if (p->next != nullptr) {
            originToClone[p]->next = originToClone[p->next];
        }
        if (p->random != nullptr) {
            originToClone[p]->random = originToClone[p->random];
        }
    }
}
```

后面会遇见图 / 树的克隆，需要用到递归，链表也可以用递归来写，如下：

```C++
// 记录 DFS 遍历过的节点，防止回头路
unordered_set<Node*> visited;
// 记录原节点到克隆节点的映射
unordered_map<Node*, Node*> originToClone;

// DFS 图遍历框架
void traverse(Node* node) {
    if (node == nullptr) {
        return;
    }

    if (visited.count(node) != 0) {
        return;
    }

    // 没有经过的点，开始 DFS
    visited.insert(node);
    // 前序位置，克隆节点
    // 由于有 random 这一条路，因此这里需要检查这个节点是否被创建过
    if (originToClone.count(node) == 0) {
        originToClone[node] = new Node(node->val);
    }
    
    Node* cloneNode = originToClone[node];

    // 递归遍历，然后后续位置构建图结构
    traverse(node->next);
    cloneNode->next = originToClone[node->next];

    traverse(node->random);
    cloneNode->random = originToClone[node->random];
}
```

其实，`originToClone`这个`map`可以充当`visited`的作用，提高效率；

### [133.克隆图](https://leetcode.cn/problems/clone-graph/)

无向连通图的深拷贝。还是通过 DFS 遍历图，然后前序遍历的位置将这个节点的clone创建出来，在后续遍历的位置构造这个节点的关系。

```C++
unordered_map<Node*, Node*> originToClone;

void traverse(Node* node) {
    if (node == nullptr) {
        return;
    } 

    if (originToClone.count(node)) {
        return;
    }

    Node* clonedNode = new Node(node->val);
    originToClone[node] = clonedNode;

    for (Node* neighbor : node->neighbors) {
        traverse(neighbor);
        clonedNode->neighbors.push_back(originToClone[neighbor]);
    }
}
```

### [242.有效的异位词](https://leetcode.cn/problems/valid-anagram/)

本质上考察的是，如何找到一种编码方法，让所有异位词的编码都相同。

一种方式是对词进行排序，但是这种时间复杂度较高。因此我们就统计每一种字符出现的次数即可。

```C++
vector<int> encode(string s) {
    vector<int> count(26, 0);
    for (char c : s) {
        vector[c - 'a']++;
    }
    return count;
}

bool isAnagram(string s, string t) {
    vector<int> count1 = encode(s);
    vector<int> count2 = encode(t);
    // 确保两个字符串中出现的字符数量都相同
    if (int i = 0; i < count1.size(); i++) {
        if (count1[i] != count2[i]) {
            return false;
        }
    }
    return true;
}
```

### [49.字母异位词分组](https://leetcode.cn/problems/group-anagrams/)

字母异位词分组的本质，还是要找到一种编码方式，能够将其分类。

有一种方法是将上面 count 数组改成一个长为 26 的 string，每一个位置记录一个字符出现的次数。然后比较 count 相同即可。

注意：`vector<int>` 没有 hash 方法，所以需要将其转换成一个 `string`。

编码方法如下：

```C++
string encode(const string& s) {
    string count(26, '0');
    for (char& c : s) {
        string[c - 'a']++;
    }
    return count;
}
```

然后，我们将所有具有相同编码的放在一起：
```C++
vector<vector<string>> groupAnagrams(vector<string>& strs) {
    unordered_map<string, list<string>> codeToGroup;
    for (const string& s : strs) {
        string code = encode(s);
        codeToGroup[s].push_back(s);
    }
    // 获取结果
    vector<vector<string>> res;
    for (auto& group : codeToGroup) {
        res.push_back(vector<string>(group.second.begin(), group.second.end()));
    }
    return res;
}
```

### [169.多数元素](https://leetcode.cn/problems/majority-element/)

这个题要求我们找到空间复杂度是 $O(1)$ 的方法。有一个很有技巧的方法，假设众数（我们要找的target）是正电、其余的全部是负电。这些粒子混在一起，最后一定是正电，我们模拟这个过程：

```C++
int majorityElement(vector<int>& nums) {
    // 我们要找的众数
    int target = 0;
    // 计数器，类比带电粒子中电荷数
    int count = 0;
    for (int i = 0; i < nums.size(); i++) {
        if (count == 0) {
            // 假设 nums[i] 就是众数
            target = nums[i];
            count = 1;
        } else if (nums[i] == target){
            count++;
        } else {
            count--;
        }
    }
    return target;
}
```
