# Union Find 并查集
## 动态连通性
### 几个术语
- 连通分量
- 连接操作
#### 联通关系的性质
1. 自反性
2. 对称性
3. 传递性

## 并查集API 与 概览
```C++
class UF {
public:
    // 初始化并查集，包含 n 个节点，时间复杂度 O(n)
    UF(int n);

    // 连接节点 p 和节点 q，时间复杂度 O(1)
    void union (int p, int q);

    // 查询节点 p 和节点 q 是否连通（在同一个连通分量内），时间复杂度 O(1)
    bool connected(int p, int q);

    // 查询当前的连通分量数量，时间复杂度 O(1)
    int count();
};
```

## 并查集核心原理
> 并查集的底层是森林，每一个连通分量都用一颗树作为代表
- `connected(p, q)` 只需要判断`p`, `q`所在的多叉树的根节点，如果相同，则说明`p`, `q`在同一棵树中
- `count()` 统计森林的数量
- `union(p, q)` 将 `p` 节点所在的树的根节点，接入`q`所在树根节点的下面，注意：是树的合并
